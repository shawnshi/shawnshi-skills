# Semantic Layer (Ontology)

定义了个人知识管理系统的核心标签与语义结构，确保长期数据的一致性与可分析性。

## 1. Core Dimensions (一级维度)
*   **#Strategy** (战略): 长期目标、商业模式、资源分配。
*   **#Delivery** (交付): 项目推进、产品落地、客户反馈。
*   **#Cognition** (认知): 读书笔记、思维模型、熵减复盘。
*   **#Health** (健康): 运动、睡眠、饮食、Garmin 数据。
*   **#Life** (生活): 家庭、旅行、兴趣、琐事。

## 2. Standard Tags (标准化二级标签)
请优先使用以下组合：

### 战略域
*   `#Strategy/MedicalAI`: 医疗AI、大模型、CDSS。
*   `#Strategy/Hospital`: 智慧医院、电子病历评级。
*   `#Strategy/Market`: 市场分析、竞对研究。

### 交付域
*   `#Delivery/Winning`: 卫宁相关项目。
*   `#Delivery/Consulting`: 咨询方案、汇报材料。

### 认知域
*   `#Cognition/Reading`: 阅读书目与洞察。
*   `#Cognition/Insight`: 突发灵感、深度思考。
*   `#Cognition/Reflect`: 周期性复盘（周/月/年）。

### 健康域
*   `#Health/Run`: 跑步数据。
*   `#Health/Sleep`: 睡眠质量分析。

## 3. Formatting Rules
*   **Space Independent**: 标签前后应保留空格（如 `今日完成 #Strategy/MedicalAI 调研`）。
*   **CamelCase**: 推荐使用大驼峰命名法以提高可读性。
