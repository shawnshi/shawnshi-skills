# 定位：医疗/B2B 核心业务域汇报 / 产品全景推介

**氛围：** 临床手术室般的无菌感，物理与数字世界的高效映射。

---

## 🎨 色彩纪律 (Color System)

**BACKGROUND COLOR (背景色)：**

* `#FFFFFF` (纯白无菌底) 或 `#F8FAFC` (极浅灰)
* **规则**：剔除一切无意义色块，保持画面的绝对干净与透亮。

**PRIMARY COLOR (主导色)：**

* `#0A2540` (权威深蓝) - 用于构建底层基座、系统框架与结构线。
* `#116DFF` 到 `#00D2FF` (青蓝渐变) - 仅作为“数字流血脉”，用于业务流向、数据传输的动态指引。

**ACCENT COLOR (警示隔离色)：**

* `#EF4444` (警示红) / `#F59E0B` (超限黄)
* **规则**：极其克制地使用，仅在医疗质量扣分、拦截弹窗、极高危节点等具体业务风险场景下激活。

---

## 🔠 字体排版 (Typography)

强制使用极高可读性的现代无衬线体 (如 Inter / 微软雅黑)，强调清晰的信息层级降维与医疗级的严谨性。

**视线截断标题 (H1)：**

* **字号**：28pt - 32pt
* **字重**：Heavy / Bold
* **颜色**：`#0A2540`
* **规则**：字数极少（强制 <15字），利用极高的对比度与粗重的字体，在无菌白底上强行拦截受众视线，直达核心。

**模块界定符 (H2)：**

* **字号**：16pt - 18pt
* **字重**：Semibold
* **颜色**：`#116DFF` (或加极浅色底卡包裹)
* **规则**：作为泳道图或业务流程的切片标题，像手术刀切面一样明确标示业务域的物理与逻辑转换。

**行为描述 (Body)：**

* **字号**：12pt - 14pt
* **字重**：Regular
* **颜色**：`#64748B`
* **规则**：承载详细的业务流转说明。视觉上必须绝对服从 H1 和 H2，不能抢占第一层级的注意力。

**视觉降级文本 (Note)：**

* **字号**：10pt
* **字重**：Light
* **颜色**：`#94A3B8`
* **规则**：用于极长但必须保留的合规免责声明、参考文献、次要指标注释等。彻底剥离其视觉干扰，使其“隐形但存在”。

## 👁️ 视觉逻辑 (Visual Logic)

**具象到抽象的极速映射：**

* 左侧放置物理医疗工具剪影（如听诊器、纸质病历）。
* 右侧呈现对应的数字实体（如 Agent 对话框、JSON 代码段）。
* **规则**：两者之间强制使用 `2px` 的青蓝动态虚线连接，直观展示物理世界向数字世界的高效投射。

**12 柱网格泳道图：**

* 多层级业务流强制对齐 Web 端标准的 12 栏网格（12-column grid）系统。
* **规则**：利用带有微阴影（Micro-shadow）的纯白卡片进行水平方向的物理隔离，确保各个系统泳道如无菌室般互不污染。

**复杂性聚焦法则：**

* 字数越少的核心指标，字号必须越大；
* 长篇解释文本强制缩至降级文本（Note）的 10pt 并置灰。
* **绝对禁忌**：全图不允许出现“字数极多且字号巨大”的视觉噪音区块，保持极其克制的信息密度。
