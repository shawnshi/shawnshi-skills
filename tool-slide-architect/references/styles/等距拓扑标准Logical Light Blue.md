# 定位：系统实施架构 / 信息化演进路线 / 高维业务推演

**氛围：** 高维透视，冷酷的逻辑展示，完全剥离情绪。

---

## 🎨 色彩纪律 (Color System)

**BACKGROUND COLOR (背景色)：**

* `#F1F5F9` 到 `#FFFFFF` 的纵向微梯度。
* **规则**：利用微妙的垂直渐变，模拟数字三维空间的高度感与纵深感。

**PRIMARY COLOR (主导色 / 架构深度色)：**
通过蓝色的明度变化构建三维架构的物理层级：

* `#1E3A8A` (深邃蓝) - 用于底层：数据湖仓、底层算力基座、基础设施层。
* `#3B82F6` (核心蓝) - 用于中层：中台业务逻辑、领域模型层、API网关。
* `#93C5FD` (轻盈蓝) - 用于表层：前端交互界面、移动端/Web应用层。

**ACCENT COLOR (狙击色)：**

* `#DC2626` (纯红) 或 `#EAB308` (纯黄)
* **规则**：仅作为界面截图的 2px 直角“狙击框”使用，呈现极度收敛、带有工业切割感的工程精确度。

---

## 🔠 字体排版 (Typography)

强调工程的精密感，强制弱化中文的圆润感（建议使用 SF Pro / 思源黑体 等具有几何切割感的现代黑体）。

**视线截断标题 (H1)：**

* **字号**：28pt - 32pt
* **字重**：Extrabold
* **颜色**：匹配当前所属架构层级的 **主导色**
* **规则**：作为三维空间视角的“上帝指令”，统领并镇压全局信息。

**模块界定符 (H2)：**

* **字号**：16pt - 18pt
* **字重**：Bold
* **颜色**：`#1E3A8A` (深邃蓝)
* **字距**：字符间距拉开 (Tracking: 150)
* **规则**：用于命名 2.5D 几何薄片的架构层级（如“DATA LAYER”），宽字距用于增加等距视角的空间开阔感。

**行为描述 (Body)：**

* **字号**：12pt - 14pt
* **字重**：Medium
* **颜色**：`#1E293B`
* **规则**：**全图封杀形容词！** 文本必须是强硬的“动宾结构”指令（例如：解析报文 / 渲染脏器 / 分发指令），绝对理性。

**视觉降级文本 (Note)：**

* **字号**：10pt
* **字重**：Regular
* **颜色**：纯白字体 + **ACCENT COLOR** 纯色底块
* **规则**：作为 UI 界面的精准标注 (Callout)，必须与 90° 直角折线强绑定。如同电路板上的元件参数说明，字号极小但极其锐利、扎眼。

---

## 👁️ 视觉逻辑 (Visual Logic)

**2.5D 等距空间架构：**

* 全局放弃扁平排版，所有层级强制使用标准的 30°/60° 夹角绘制 3D 几何薄片。
* 像堆叠玻璃板一样构建数字世界的三维透视沙盘，层级之间保持严格的垂直 Z 轴距离。

**界面原石与直角引出：**

* 必须插入 100% 真实的系统界面或数据仪表盘高清截图，将其作为“物理原石”嵌入架构中。
* **强制约束**：标注引线必须使用带有 ACCENT COLOR 的 90° 直角折线引出，**绝不允许使用任何柔和曲线或斜线**。
