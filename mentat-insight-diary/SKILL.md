# 内观审计日志

## 证据与隐私边界

- 只记录真实发生且能从当前会话、用户材料或获准数据源核验的事件。
- 不主动读取日历、邮件、私人日记、健康数据或其他敏感来源。需要这些材料时先说明用途并取得授权。
- 不要求每次日志都必须存在失败。没有重要事件时，直接说明本期无足够材料，不编造摩擦。
- 默认只生成草稿。写入季度档案、知识库或外部系统前，必须获得用户明确授权和目标路径。
- 删除或概括日志中的凭据、私人内容、客户秘密和不必要的个人标识。

## 执行流程

1. 确定记录时间范围、事件范围、叙述视角和是否需要持久化。
2. 建立证据清单：事件、时间、动作、结果、错误信息、相关文件或来源。将不确定项明确标记。
3. 按需读取 [assets/ooda_template.md](assets/ooda_template.md)，使用 OODA 结构组织内容：
   - Observe：记录事实和直接观测，不加入原因判断。
   - Orient：解释约束、冲突、假设和可能根因，并标明证据强度。
   - Decide：写出选择、放弃的方案和决策依据。
   - Act：记录已执行动作、验证结果和下一步触发条件。
4. 检查是否把推断写成事实、是否遗漏关键失败、是否泄露敏感信息。
5. 向用户展示草稿。用户明确要求保存时，再写入用户指定路径；未指定路径时先询问，不自行选择全局档案。
6. 用户明确要求登记长期知识时，提取最小必要的规则或结论并再次确认写入范围。

## 输出结构

```markdown
## [日期或时间范围]｜[事件标题]

### Observe
[可核验事实]

### Orient
[约束、根因假设、证据强度]

### Decide
[选择与理由]

### Act
[动作、验证、下一触发条件]

### 未决问题
[缺口与需要补充的证据]
```

## 完成检查

- 所有事件均有当前证据支持。
- 事实和解释已分开。
- 没有强行制造失败或过度归因。
- 敏感信息已最小化。
- 未经授权没有读取私人来源或执行持久化。
