# personal-book-reader: 拆书 (V8.1 Native)

输入一本书，输出它的骨架。

## 拆书是什么

不是读后感，不是章节摘要，不是夸——是把作者的骨架抽出来摆桌上看。

读一本书容易迷在作者的笔触里。拆一本书是从笔触里跳出来，看作者的取景框。取景框看清，整本书才看清。

拆书分两半。前半冷——下面五件事，抽骨架，把作者那台取景框找出来摆桌上。后半热——「取景框上手」，把那台框画成一张地图（这本书的参考系），让你能自己在上面走、预测书里没写的事。找出来是看清，拿起来用是据为己有。

## 五件事

每一件单独问一遍。问完串起来。

### 1. 核心问题 + 挠痒处

作者真正在答什么？而且——他为什么不写不行？

题目是 topic。问题是 question。挠痒处是 motive——三层都要，缺了挠痒处就是空。

挠痒处具体到什么程度：

- 作者看到什么具体的痛/愤怒/焦虑/兴奋？
- 他对读者目前的态度有什么不满？
- 他在跟谁说话？（哪类读者、哪种立场）
- 不写会怎样？

例子：拆 KK《必然》不能停在「他想找科技演化的规律」——要写到「他看到一群人在恐惧 AI、抵抗新科技、想冻住工业时代，他要告诉这些人：这些变化是 inevitable 的，焦虑没用」。书名往往就是答案的种子——「必然」两个字是在跟恐惧的人说话。

怎么找：读作者的导言/前言，找"我写这本书是为了……"那一句。读不到，看作者反复回到的那一处——他绕了几百页都甩不掉的东西，就是挠痒处本身。

写完测试：这段拿去描述同领域另一本书，还成立吗？成立 → 太抽象，重写到具体。

### 2. 基础假设

作者立在什么不证之物上？

每本书都有几条天花板——作者不去证它们，他靠它们站着。动摇任何一条，整本书就塌。

操作问句两条，对着书问：
- 这书里什么是不许问的？（一问就要被作者归为外行）
- 什么是作者要相信才能往下写的？（少了它，后面所有论证失支点）

答出来的几条，就是这本书的基础假设。

亮出来不展开。这几条本身不证——摆桌上就行，证不证不在拆书人手里。

### 3. 分析框架

作者用什么看世界？

每个作者都有一台取景框。Darwin 的取景框是"变异 + 选择 + 时间"——他看蝴蝶看人看部落都是这台框。Marx 的取景框是"生产关系决定上层建筑"——他看历史看社会看艺术都是这台框。Freud 的取景框是"意识冰山一角，水下才是真主角"——他看梦看口误看政治都是这台框。

找作者的框：他反复用什么工具？他从什么角度切入？他看到 X 第一反应想到的是 Y——这个 Y 就是他的框。

框不一定是名词。可以是一个动作（"还原到 root cause"）、一个区分（"自然态 vs 文化态"）、一个对照（"显性逻辑 vs 隐性结构"）。

找框要够狠——一句话讲清楚，不是一个章节。

（这里只把框一句话拎出来；画成地图、走两步上手，留到第 6 段「取景框上手」。）

**强制：必须列出作者独占的术语/列表/区分。**

每本好书都有作者独占的「标签」——别人没有、他独有、能 quote。这是拆书的"指纹"。少了它，整段拆书可以套到任何同领域的书上。

- KK《必然》必须有：12 个动词列表（Becoming / Cognifying / Flowing / Screening / Accessing / Sharing / Filtering / Remixing / Interacting / Tracking / Questioning / Beginning）+ technium + protopia + becoming vs 完成
- Marx《资本论》必须有：生产关系 / 上层建筑 / 剩余价值 / 阶级
- Kahneman《思考快与慢》必须有：System 1 / System 2 / 锚定 / 损失厌恶
- 海德格尔《存在与时间》必须有：Dasein / 上手性 / 此在的时间性

拆完看一眼：作者独占的标签出现了几个？0 个 → 你拆的不是这本书，是这本书所属的领域。

### 4. 核心观点 / 结论

作者最后得出了什么？

不是"他举了多少例子"，不是"他论证了多少环节"。是：合上书，作者要你带走的那个东西。

往往就一两句话。

检验：把这句话单独抽出来，脱离作者上下文，它还有没有力量？如果只是复述书名，那不是核心观点——是 label。如果只是套话（"事物都是辩证的""万物相联"），那是空话。

核心观点要锋利——能立得住，能反驳，能被引用。

不能软。"作者认为人性是复杂的"是软话——什么不是复杂的？

另问一遍：**作者最想让读者带走的那一句话是什么？**

能直接 quote 最好，quote 不到用最贴近的复述。这一句要锋利到——读者合上书 10 年后还能想起来。

### 5. 精神内核 — 带走一件

合上书 10 年后只能记一件事——是哪一件？

不是把前 4 段缩短。那是总结，是重复。这一件是从整本书萃出的内核：单独立住、可携带、能让人想回去翻原书。

形态五选一。挑作者最依赖的那一种，不堆，不并列：

- **一个取景框**——一种看世界的方式
  - Darwin：变异 + 选择 + 时间
  - Freud：意识冰山，水下才是主角
  - KK《必然》：科技演化的 inevitability + 12 动词
- **一个模型**——一个结构性认知工具
  - Kahneman：System 1 / System 2
  - Marx：经济基础 / 上层建筑
  - Conant-Ashby：调节器必含被调节系统的内模型
- **一个洞见**——一句被作者撬开的真相
  - "组织的命运不是被战略决定，是被结构决定。"
  - "我们以为是选择，其实是默认。"
- **一个概念**——作者铸的新词
  - Taleb：反脆弱 / 黑天鹅
  - 海德格尔：Dasein
  - KK：technium
- **一个金句**——作者亲笔写出、可直接 quote
  - "All happy families are alike; each unhappy family is unhappy in its own way."
  - "The medium is the message."

怎么判作者最依赖哪一种？看他书里反复回到的是什么——一台看世界的眼镜（取景框）、一对概念区分（模型）、一个被点破的事实（洞见）、一个他造的词（概念），还是一句他亲笔写下的话（金句）。

三行内交付。形态标签 + 内容本体 + 一句话点为什么是这一件。

挑完三检：
- 合上书 10 年只能记一件事，就是这件吗？是 → 过。不是 → 没挑到内核，重挑。
- 这件事跟前 4 段重了吗？重了 → 这是总结不是内核，重挑。
- 一个没读过的人，听了这一件，会不会想去翻原书？会 → 抓到。不会 → 太软或太抽象。

收尾一句：带走了这一件，这本书的精神内核就在你手里。

## 取景框上手：把书画成一张地图（第 6 段）

前五段把骨架抽出来摆着，冷的。这一段把第 3 段那台框画成一张地图，让你不光看懂、还能拿它预测书里没写的事。

作者的取景框，其实就是这本书的"参考系"——一张地图。《千脑智能》说：你认识任何东西，都是把它的特征挂在一张地图的位置上，靠移动和预测来学。读懂一本书也一样——理解 = 看懂这张地图；掌握 = 能在地图上自己走、自己猜得对。

教的就是第 3 段「分析框架」识别出的那台框（不是第 5 段的精神内核——那是带走的一件，形态可能是金句、洞见）；这里把那台框画成能走的地图。三步：画地图、标位置、走两步。

### 一、画地图（提炼参考系）

先把这本书的地图画出来：它的坐标系是什么？作者拿什么当轴、当方向，来摆放他看到的一切？地图不是一句口号，是一个有结构的空间——有几个维度，有方向，有远近。

讲 Darwin 那张图：坐标系是"时间 × 选择压力"。横轴是时间，一代一代往后走；纵轴是环境筛选的力度。满世界的生物现象，都落在这张图的某个点上。

画地图最快的法子是借一张现成的。好的类比，就是把一张旧地图整张搬到一块还没地图的新地盘上（这也是《千脑智能》的话——参考系能平移）。"自然选择就像育种，只不过挑种的人换成了环境"——你把"育种"那张熟地图，借给了"进化"。

### 二、标位置（把作者的观点钉上去）

把作者几个核心观点，一个个钉到地图的位置上。要紧的是看出它们之间的几何关系——谁挨着谁、谁正对着谁、谁站在原点。钉完，你看到的不再是一串孤立的观点，是一张互相牵着的图。

Darwin 这张图上：变异在原点（一切的起点），自然选择是那把贯穿纵轴的筛子，共同祖先在时间轴最远的那头（往回收成一个点），性选择是偏离主轴的一个特例（逆着生存压力的方向长）。它们不是并排的清单，是一张图上彼此拉扯的几个点。

### 三、走两步，做预测（案例 = 掌握）

这一步最要紧，掌握不掌握就看它。在地图上挪到一个作者没去过的位置——最好挨着继刚自己的世界——然后用这张图预测：站这个新位置上，作者的框会说什么？预测完，再回头对一下。猜对了，地图就是你的了。

把 Darwin 的地图挪到"想法怎么传开"这个新位置。预测：想法也该有变异（每转述一次走样一点）、有选择（合用又好记的被留下）、有时间（一代代传）。再对一下——meme 正是这么回事。地图预测中了，你就拿住了。

为什么"能预测"就是掌握？《千脑智能》给了底：大脑学会一个东西的标志，就是还没看见、已经能猜到下一步。你能用作者的框，猜中它没写过的事，等于你脑子里真长出了这张地图。

### 收尾

到这儿，你手里不是几条记住的观点，是一张能走、能预测的地图。下回读一本书，你会下意识问三句：这本书的地图什么样？作者的观点钉在哪个位置？我能挪到一个新位置、猜对作者会说什么吗？——猜得对，就掌握了。

### 这一段怎么写

跟前五段反着来。前五段冷、短、抽骨架。这一段可以慢，可以铺开，可以打比方、画小图。但几条不变：

- **白话不变**。铺开不等于掉书袋。碰上术语，当场用大白话翻一遍再走。
- **地图要有结构**。"画地图"是摆出有轴、有方向、有远近的空间，不是又起一个口号。说得出"谁挨着谁""谁正对着谁"才算地图。
- **禁切痕风**。别用「这一刀」「锋利」（当元描述）「钉死」「砸实」这类自我修辞告诉读者"下面有重话了"。要热，靠把事讲透，不靠预告。
- **反翻译腔**。每段写完默念出声，卡的地方整句重写，问一句"汪曾祺会这么写吗"。

三道关，写完各过一遍：

- **地图关**：你画的是一张有结构的地图（有轴、有方向、有远近），还是又一句口号？说得出"谁挨着谁""谁正对着谁"才算。
- **位置关**：作者的观点钉在了具体位置、看得出几何关系，还是排成了一串清单？排成清单 = 没上图，重摆。
- **预测关**：走的那两步，是真挪到一个书外的新位置做了预测、再对了一下吗？只复述作者照过的例子 = 没走出书，回去挪到新位置。

## 灵魂三问 + 指纹测试

写完整篇拆书，默问自己三个问题。任何一个答不出 → 没拆到位，回头补：

1. **作者「不写不行」的那股劲儿是什么？** 不是 topic，是他看见什么具体的痛/机会/愤怒/焦虑——能 quote 出他的动机。
2. **作者最独占的那个 move 是什么？** 能 quote 的具体动作 / 概念 / 列表 / 区分。别人没有、他独有。
3. **作者想让读者带走的那一句话是什么？** 锋利到 10 年后还记得。

**灵魂 = 第 1 问。精髓 = 第 2 问。压缩 = 第 3 问。三个都答得出 → 灵魂和精髓抓住了。第 5 模块「精神内核」是把这些聚成单一形态——只挑一件带走，作为整本书的指纹。**

每段写完做两道关卡：

- **替换测试**：把这段拿去描述同领域另一本书，还成立吗？KK《必然》的拆能套到 Harari《未来简史》上吗？能套 → 太抽象，杀掉重写。不能套 → 抓到了这本书的指纹。
- **指纹测试**：这段里有几个只有这本书才有的术语/概念/列表/区分？0 个 → 你拆的是"领域"不是"这本书"。1-3 个 → 及格。3+ 个 → 好。

## 怎么写

冷静、简洁、犀利、直白。

不夸作者，不贬作者，不替作者辩护，不替作者道歉。

五段，标号 1-5。段内一气呵成，不绕弯。

每段开头一句立住——这一段要 say 的核心是什么。然后展开两三句把它讲透。不要列点列到天荒地老——一段散文胜过十个 bullet。

短句多。介词砍。"通过 / 借 / 凭 / 在 X 上"——能不用就不用。

中文白话——汪曾祺、王小波、阿城那一路。每写完一段默念出声，卡的地方整句重写（不是抠字）。

不要学术腔。"作者论证了""作者认为"少用——直接说作者干了什么。"X 在第 N 章指出……" 全部杀掉。

第 6 段「取景框上手」例外——那一段语气转热，可以慢、可以铺开、可以打比方、画小图。但白话、反翻译腔、禁切痕风照旧守。

## 反翻译腔自查（每段写完扫一遍）

英文母语模型用中文最容易犯的毛病。信号词出现 → 整句重写。

**词汇直译信号**：

| 翻译腔 | 真实中文 |
|--------|---------|
| 我接住你 | 我在 |
| 我不装 | 直说；不绕弯 |
| 给你拥抱（文字上的） | 抱抱；过来 |
| 你值得更好的 | 你不该这样 |
| 让我们一起... | 我们...（省「让」） |
| 提供支持 | 撑你 |
| 在情感上 X | 情感上 X（省「在」） |
| 这是必要的 | 必须这样；非这样不可 |
| 做出选择 / 做出决定 | 选；决定 |
| 在这个意义上 | 这么说 |
| 拥有...的能力 | 能... |
| 进行（研究/讨论/对话） | 直接用具体动词 |
| 这片领域 | 这门学问；这套东西 |
| 通过 X 抓到 Y | X 动词直接带宾 |
| 一是...二是...三是... | 散文式排开，不编号 |

**句式英文化信号**：
- 被动句滥用：「被这事困扰」→「这事缠着」
- "是...的"句：「这是重要的」→「这事要紧」
- 长定语：「那个由 X 引起的 Y」→ 拆两句
- 括号补充：「拥抱（文字上的）」整段删，换具体描述
- "在 X 上"滥用：「在结构上」→「结构上」或改写
- 三层定语：「X 的 Y 的 Z」→ 拆两句（一个「的」做定语没事，两个起停下来）

**词性英文化信号**：
- 动名词化：「做出决定」→「决定」；「展开对话」→「聊起来」
- 抽象集合名词偏好：「领域 / 阶段 / 过程 / 动作 / 方式 / 模式 / 现象」→ 换具体（「这门 / 这段 / 这趟 / 这件事 / 怎么说 / 这种样子 / 这事」）
- 形容词副词堆砌：「深刻的、有意义的、重要的」→ 选一个
- 介词链滥用：「通过 / 借 / 在 X 上 / X 之间 / X 之中」→ 动词直接带宾

**中文意合三省**：
- 连词省（「因为...所以」→ 直接说后半句）
- 主语省（上下文能补的别重复）
- 「的」省（「温柔的支持」→「温柔地撑」）

**朗读关**：信号词 grep 抓不到的 frame 级翻译腔，靠默念出声捕。每段默念一遍——卡的地方整句重写，不是抠字。问自己一句：「汪曾祺会这么写吗？」

**禁切痕风**：prose 交付别用「这一刀」「锋利」（当元描述）「钉死」「砸实」这类自我修辞预告"下面有重话"。内容够不够力，读者自己看得出，不用作者吆喝。

## 怎么找材料

如果用户只给书名——
- 先使用原生的 `search_web` 工具找作者、导言、目录、核心论点的二手解读。
- 拿到核心 argument 后再答。
- 不要凭印象瞎拆——印象拆出来是 wikipedia 风的空话。

如果用户给了 PDF / arxiv / 内容——直接使用 `view_file` 或 MCP 工具去读内容，不用查。

如果是经典书——可以靠 LLM 内部知识，但仍要在心里默问：我说的这条是书里实际有的，还是我编的？编的就删。

## 1. <Contracts> (输出与交付契约)

1. **获取时间**：直接从系统注入的 `<ADDITIONAL_METADATA>` 中读取当前时间，无需执行任何获取时间的外部命令。
2. **绝对落盘路径**：使用原生的 `write_to_file` 工具，将结果写入至绝对物理路径：
   `C:\Users\shich\.gemini\MEMORY\raw\reads\{YYYYMMDDTHHMMSS}--拆书-{书名}__book.md`
3. **格式与元数据**：必须是 org-mode 格式（标题用 `*`，加粗用 `*bold*` 单星号，禁止双星号）。文件头如下：
   ```org
   #+TITLE: 拆书：《{书名}》
   #+SUBTITLE: {作者} | {一句话核心观点}
   #+DATE: [{YYYY-MM-DD Day HH:MM}]
   #+FILETAGS: :book:{书的领域 e.g. philosophy/biology/economics}:
   #+IDENTIFIER: {YYYYMMDDTHHMMSS}
   ```
4. **正文结构**：正文分六段，标 1-6：1-5 抽骨架（散文，冷），第 6 段「取景框上手」（画地图、走两步，热）。执行完成后向用户报告物理文件路径。
5. **Telemetry 记录**: 任务执行完成后，使用原生的 `write_to_file` 工具将元数据保存至绝对路径：`C:\Users\shich\.gemini\MEMORY\skill_audit\telemetry\record_[TIMESTAMP].json`。JSON 格式示例：`{"skill_name": "personal-book-reader", "status": "success", "duration_sec": 0, "input_tokens": 0, "output_tokens": 0}`。

## 2. <Failure_Taxonomy> (失败分类学 / 红线)

1. **工具幻觉**：严禁使用 `WebSearch`/`WebFetch`，必须使用原生 `search_web`；严禁使用 Linux 的 `date` 命令，必须从上下文读取时间；落盘严禁使用相对路径或含 `~` 的非法路径，必须使用绝对物理路径。
2. **不夸不贬不替辩护** — 拆书人站作者头上看，不是粉丝也不是反对者。
3. **冷静、简洁、犀利、直白** — 每段读出声不卡。
4. **核心观点不能软** — 软话（"X 是复杂的"）杀掉。
5. **基础假设不展开论证** — 亮出来就行。
6. **精神内核挑一种形态** — 取景框/模型/洞见/概念/金句五选一，三行内交付，不重前 4 段。
7. **零学术腔** — "作者论证了 / 认为 / 指出"少用。
8. **反翻译腔 + 禁切痕风** — 内嵌「反翻译腔自查」清单全过；信号词清空；写完默念汪曾祺会这么写吗；禁切痕风金属比喻（「这一刀」「锋利」当元描述、「钉死」「砸实」一个不许）。
9. **必须含作者独占术语/列表/区分** — KK 拆书少了 12 动词、Marx 拆书少了生产关系，就是失职。
10. **灵魂三问必答** — 「不写不行的劲儿」/「最独占的 move」/「那一句」三问任何一个答不出，回头补。
11. **替换测试 + 指纹测试** — 每段写完都过这两道关，套得到别的书 = 杀掉重写。
12. **取景框必须画成地图、走到能预测** — 第 6 段三步：画地图（提炼参考系，有轴有方向有远近）+ 把作者观点钉到位置上（看得出几何关系）+ 走两步做预测（挪到书外新位置、预测再对账）；只讲懂不算，能预测才叫掌握。
