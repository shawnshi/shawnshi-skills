# 跨领域结构映射

## 提取源域结构

1. 明确现实问题、目标、参与者、资源、状态、约束和反馈。
2. 用对象和关系描述结构，保留时间、方向、容量和不可逆性。
3. 把事实与解释分开。没有形式证明时使用“结构相似”或“可检验映射”，不要声称同构。

## 选择目标域

1. 选择一至三个有成熟机制且与源域存在结构对应的领域。
2. 按需读取 [references](references) 中与目标域直接相关的文件，不要一次加载全部领域资料。
3. 需要查看领域目录时，可在不修改文件的模式下使用 [scripts/domain_manager.py](scripts/domain_manager.py)。
4. 新增或改写领域知识文件属于持久化变更，必须先获得用户明确授权；不要自动生成领域框架或安装依赖。

排除只提供情绪感染力、无法给出可验证机制的类比。对争议理论标明证据状态。

## 建立与检验映射

为每个候选映射列出：

| 源域 | 目标域 | 对应关系 | 保留的约束 | 失真点 |
|---|---|---|---|---|

然后执行：

1. 在目标域内应用具体机制或定理，写出必要前提。
2. 把推论逆映射到源域，转化为可执行动作。
3. 检查交换性：不同路径是否得到兼容结果。
4. 寻找反例、边界条件和量纲不一致。
5. 用小实验、指标或决策阈值验证映射，而不是依赖修辞。

任务包含彼此独立的多个目标域且当前环境支持时，可选用子代理并行分析；主代理使用同一源域结构和检验标准比较结果。

## 交付

输出：

1. 源域结构；
2. 目标域及选择理由；
3. 对应表和机制推演；
4. 逆映射后的方案；
5. 映射失效条件；
6. 最小验证实验；
7. 与常规方案相比新增的真实变量。

默认在当前对话交付。写入知识库、保存新领域文件或持久化洞察必须由用户明确授权并指定位置。
