# 输出格式规范 (Output Format Reference)

> 所有模式的审计报告输出标准。代理输出必须严格遵循对应模式格式。

---

## 一、Quick 速查模式

```markdown
## Quick Council: [议题]

### 各方视角

**🏛️ Architect (Serena):**
[30-50字简要观点]

**🎨 Designer (Aditi):**
[30-50字简要观点]

**⚙️ Engineer (Marcus):**
[30-50字简要观点]

**🔍 Researcher (Ava):**
[30-50字简要观点]

### 快速判定
**共识:** [是否一致？在什么方面？]
**风险:** [发现的红旗？]
**建议:** [继续推进 / 重新考虑 / 升级至完整辩论]
```

---

## 二、Consensus 共识模式 (Three-Round Debate)

```markdown
## Council Debate: [议题]

### 第一轮：立场陈述
**🏛️ Architect (Serena):**
[50-150字 — 从架构视角的初始立场]

**🎨 Designer (Aditi):**
[50-150字 — 从设计视角的初始立场]

**⚙️ Engineer (Marcus):**
[50-150字 — 从实现视角的初始立场]

**🔍 Researcher (Ava):**
[50-150字 — 基于数据/先例的立场]

### 第二轮：挑战与回应
**🏛️ Architect (Serena):**
[50-150字 — 引用其他成员的具体观点进行回应]

[...其他成员同上...]

### 第三轮：综合
**🏛️ Architect (Serena):**
[50-150字 — 共识区域、持续分歧、最终建议]

[...其他成员同上...]

### Council 综合判定
**共识区域:**
- [3+ 代理一致同意的观点]

**残余分歧:**
- [仍有争议的观点]

**推荐路径:**
[基于论据权重的综合建议]
```

## 三、Validation 验证模式 (Adversarial Refinement)

```markdown
## Adversarial Validation: [议题]

### 第一轮：竞争方案
**方案A (务实派):** [优先实现速度与快速见效]
[完整方案描述]

**方案B (理想派):** [优先最佳实践与长期质量]
[完整方案描述]

**方案C (怀疑派):** [优先风险控制与故障预防]
[完整方案描述]

### 第二轮：残酷批判
**✅ 各方案的优势:**
- 方案A: [具体优势]
- 方案B: [具体优势]
- 方案C: [具体优势]

**❌ 各方案的缺陷:**
- 方案A: [具体缺陷]
- 方案B: [具体缺陷]
- 方案C: [具体缺陷]

**💊 没人愿意听的真相:**
[所有方案都回避的核心问题]

### 第三轮：协作综合
[取各方案最强元素的统一解决方案]

**保留的权衡:**
- [诚实承认未完全解决的张力]
```

## 四、Attack 攻击模式 (32-Agent Parallel Analysis)

```markdown
## Red Team Report: [被攻击论点]

### Phase 1: 原子化分解
**核心命题:** [一句话]
**24 条原子断言:**
1. [断言1]
...
24. [断言24]

### Phase 3: 32 代理汇聚分析
**🔴 致命弱点 (5+ 代理共识):**
- [弱点1] — 引用 philosophy.md 公理编号
- [弱点2]

**🟡 显著弱点 (3-4 代理共识):**
- [弱点]

**🟢 独到洞察 (1-2 代理):**
- [洞察]

### Phase 4: 钢人论证 (Steelman)
**最强版本:** [一句话 — 对方论点的最佳表述]

1. [12-16字 — 最有力的开场]
2. [12-16字 — 有力证据]
3. [12-16字 — 历史先例支持]
4. [12-16字 — 切实关切]
5. [12-16字 — 批评者错在哪]
6. [12-16字 — 忽略的真实风险]
7. [12-16字 — 聪明人为何相信]
8. [12-16字 — 必须认真对待的理由]

### Phase 5: 反驳裁决 (Counter-Argument)
**立场:** [一句话]

1. [12-16字 — 根本缺陷]
2. [12-16字 — 核心弱点]
3. [12-16字 — 历史反例]
4. [12-16字 — 隐藏假设]
5. [12-16字 — 反例/例外]
6. [12-16字 — 被忽略的因素]
7. [12-16字 — 二阶效应]
8. [12-16字 — 致命结论]

**最终评定:** [经过完整分析后对论点根本健全性的判断]
```

## 五、通用质量标准

| 维度 | 要求 |
|------|------|
| **引用** | 质疑必须引用 `philosophy.md` 中的公理/谬误/偏见编号 |
| **语言** | 禁止使用形容词修饰，纯因果逻辑 |
| **互动** | 第二轮起必须引用其他代理的具体观点 |
| **钢人** | 攻击前必须先构建最强防御版本 |
| **诚实** | 残余权衡必须明确承认，不能假装已解决 |
