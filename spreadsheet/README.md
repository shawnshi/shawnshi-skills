> **核心内核**：重装级 Excel 治理系统。严禁硬编码，强制推行“公式化逻辑”与“行业标准颜色编码”，构建可审计的专业模型。

## 0. 本质与边界 (Essence & Boundary)
- **核心定义**: 数据逻辑的守门员，负责确保所有 Excel 模型都具备“自重算”能力与极高的透明度。
- **反向定义**: 它不是一个简单的表格修改器，而是一个具备财务级严谨性的数据架构引擎。
- **费曼比喻**: 它像是在你的 Excel 表格里装了一台“逻辑显微镜”，它能确保所有的数字都是算出来的，而不是你随手填进去的。

## 1. 生态位映射 (Ecosystem DNA)
- **MSL 契约**: 处理“单元格依赖”、“逻辑算式”、“场景假设”等实体。
- **ACE 角色**: 作为 **Data Auditor (数据审计者)**。

## 2. 逻辑机制 (Mechanism)
- [Data Ingestion] -> [Formula Construction] -> [Recalculation Scan] -> [Visual QA]

## 3. 策略协议 (Strategic Protocols)
- **硬编码熔断**：严禁在逻辑计算单元中直接填入数字，所有参数必须由 Assumption 单元格驱动。
- **颜色编码强制**：蓝色代表输入，黑色代表计算，绿色代表引用，红色代表外链。
