# 平面设计提示词

## 执行流程

1. 从用户输入提取主题、载体、受众、情绪、画幅、必须出现的元素、禁用元素和文字要求。
2. 用户已提供完整提示词或要求原样使用时，保留其语义与约束，不擅自添加流行修饰词。
3. 用户只给出简短主题时，补足以下视觉参数：
   - 核心象征和视觉焦点；
   - 构图、负空间与层级；
   - 有限色板、印刷质感和线条语言；
   - 标题位置、字体类别与可读性要求；
   - 画幅、留白、安全区和排除项。
4. 需要风格参考时，按需读取 `references/` 下的美学、构图、封面与类型模板。示例图片只用于理解构图，不要复制其中受保护的具体内容。
5. 用户要求模仿具体创作者时，优先转写为可观察的视觉特征、时期、媒介和构图语言，不声称复制其独特风格。
6. 用户只要提示词时，交付可直接使用的提示词和必要的负面约束，不调用图像生成。
7. 用户明确要求图片时，使用当前图像生成能力直接生成。不要仅因提示词较短而暂停确认；只有缺少必须的源图或关键约束且无法合理推断时才询问。
8. 用户要求编辑现有图片时，先确认目标图片可用并检查其内容，再执行编辑。
9. 不自动保存提示词偏好、写入知识库或创建临时文件。

## 可选脚本

需要确定性枚举现有风格或生成结构化提示词时，可先查看并运行 `scripts/generate_mondo.py`。不要自动安装依赖；脚本输出需要人工检查后才能作为最终提示词。

## 提示词结构

按实际需要组合，不机械填充：

```text
[设计载体与主题]；[核心象征]；[构图和负空间]；[色板]；
[平面媒介与纹理]；[文字和排版要求]；[画幅与安全区]；
避免：[写实摄影、复杂三维效果、无关物体、不可读文字等]
```

## 完成检查

- 主题、焦点和构图没有互相冲突。
- 画幅与发布载体匹配。
- 文字要求具体且不承诺模型一定准确拼写长文本。
- 未引入用户没有要求的品牌、人物或受保护内容。
- 用户请求生成图片时已直接进入生成流程。
