# 灵魂句与七拍故事弧：手艺人操作手册

## 1. 灵魂句炼金术（Title 写作的 Few-Shot）
title 是这篇笔记的*灵魂句*——读者扫一眼就知道这篇论文带走什么。抽出 title 贴墙上，它本身就该像一句警句。

**写作自查（必过）**
- [ ] 像不像一句篇名/警句？
- [ ] 删掉任何一个字会不会塌？汪曾祺会这么写吗？
- [ ] 包含英文术语了吗？（必须全中文，人名、特定产品名除外）

**正例对照（核心范本，仔细体会右侧的“中文凝练”与动词使用）**
| 论文核心思想 | ✗ 翻译腔 / 口语化 | ✓ 中文凝练 |
|------------|------------------|-----------|
| 奖励信号把模型锁在已会轨迹里，擦掉题目就解锁 | 奖励信号会把模型锁死在已会的轨迹里 | 学会，反成枷锁 |
| 只用错样本做 RL，反思能力自己长出来 | 只用错样本做 RL，反思能力自己长出来 | 错处长出反省 |
| 模型在向量空间里思考比生成 token 更省更准 | 模型在向量里想，比一边想一边写更省 | 默想胜出口 |
| 多智能体缺的是组织协调而非个体智能 | Multi-agent 缺的不是聪明，是 HR | 多智不如善织 |
| 写作时预测剩余长度让模型知道何时收尾 | 模型每写一个 token 就知道离收尾有多远 | 知止方能落笔 |

**可识别性兜底**
如果凝练的标题让外行完全猜不到方向，必须用中文副标题兜底解释（如 `#+subtitle: 把"还要写多远"做成一个 value 函数`）。

## 2. 七拍故事弧落地指南
这不是七个子标题，是节奏要求，全部挂载在同一个“锚点（例子）”上。

- **问题：搭台**：必须包含拍1(亲历)+拍2(困境)+拍3(旧路)。用一段连续叙述让读者**亲历**一个失败的例子（如一道具体的错题）。不要说“之前存在什么短板”，要让读者亲眼在这个例子上看旧路是怎么撞墙的。
- **翻译：推进**：拍4(转折)+拍5(解法)+拍6(结局)。必须接着“问题”里的那个**具体例子**接着推演，换例子 = 切地图 = 认知断层。
- **承重类比**：不只是装饰，要能映射方法的关键组件。例如：把模型映射成跑步选手，value head 映射成手表。
- **变形替代定义**：解释新概念或关系时优先用“把 A 变形成 B”，不要枯燥地说“A 和 B 是 XX 关系”。
- **禁止公式裸奔**：主文中严禁直接放置原始 LaTeX 公式。必须翻译为外行语言（如：“意思是模型每写一字都被监督一次”）。
- **3个故事道具**：核心概念必须至少拆解3个，且每个道具都必须带回到最初那个例子上落地，解释“少了它在这个例子里会怎样”。
