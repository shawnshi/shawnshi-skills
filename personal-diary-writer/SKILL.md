# 日记写入

## 写入前

1. 确认用户明确要求持久化，而不是只想生成或润色草稿。
2. 确认待写内容、日期、目标文件和写入方式（追加、前置或新建）。目标不明确时先询问，不猜测默认位置。
3. 只保存用户提供或已确认的事实。缺失字段可以省略或标为“未提供”，不得自行读取日历、健康设备、历史对话或其他私人数据。
4. 若内容含健康、财务、身份或第三方隐私信息，提示将写入的字段和位置，并取得明确确认。
5. 写入前展示最终文本。若用户已经逐字提供内容、明确目标位置和写入动作，可直接执行；其他情况等待确认。

## 写入

1. 使用当前环境可用的原子编辑方式，保留目标文件原有编码和换行风格。
2. 修改现有文件时只变更目标日期或指定区块，避免覆盖无关内容。
3. 写入失败时停止并报告实际错误；不要改写其他路径作为静默回退。
4. 写入后重新读取目标区块，核对日期、内容和重复项。

## 推荐格式

仅保留有实际内容的章节：

```markdown
# YYYY-MM-DD 星期X

## 今日工作
- ...

## 核心产出
- ...

## 明日安排
1. ...

## 复盘
...

## 身体状态
- 数据来源：...
- 已确认指标：...
```

健康指标必须注明来源和采集时间；用户未提供时不要生成分数、归因或干预指令。

## 边界

- 不把“帮我写一篇日记”视为磁盘写入授权；先交付草稿。
- 不自动同步到知识库、云端、日历或健康平台。任何第二处持久化都需要单独授权。
- 不保存隐藏推理、访问凭证、令牌或无关私人数据。
- 不为了满足固定模板而补造事件、情绪、生理指标或结论。
