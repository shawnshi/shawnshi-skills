# Domain: 控制系统
# Source: Wiener《控制论》, Ashby《控制论导论》, Ogata《现代控制工程》
# Structural_Primitives: 反馈、调节、稳定、误差、响应、振荡

## Core Objects
- 系统: 被控制的实体，具有输入和输出
- 控制器: 根据误差产生控制信号的组件
- 传感器: 测量系统输出的装置
- 执行器: 实施控制动作的装置
- 设定点: 期望的系统状态或输出值
- 误差: 设定点与实际输出之间的差异

## Core Morphisms
- 反馈: 将输出信息返回输入端进行比较
- 调节: 根据误差调整控制动作
- 放大: 增强控制信号以驱动执行器
- 延迟: 信号传输和处理的时间滞后
- 振荡: 系统在设定点附近的周期性波动
- 阻尼: 抑制振荡的机制

## Theorems / Patterns

### 1. 负反馈稳定原理
**内容**: 负反馈（将输出与期望值的差异反向作用于输入）是系统稳定的核心机制。适当的负反馈可以抑制扰动，维持稳态。

**Applicable_Structure**: 自我调节、偏差纠正、稳态维持、自动平衡

**Mapping_Hint**: 当 Domain A 需要维持某个指标稳定时，建立负反馈回路——检测偏差，反向调节，形成自我调节机制。

**Case_Study**: 恒温器：温度高于设定值时减少加热，低于设定值时增加加热，维持温度稳定。

### 2. 反馈延迟与振荡
**内容**: 反馈回路中的延迟会导致系统振荡。延迟越长，振荡周期越长，系统越难稳定。

**Applicable_Structure**: 调控时机、过度反应、周期性波动、时滞效应

**Mapping_Hint**: 当 Domain A 出现周期性波动时，检查反馈延迟。缩短反馈回路或降低反馈强度可以减少振荡。

**Case_Study**: 淋浴水温调节：如果等待太久才调节，会反复在过热和过冷之间振荡。

### 3. 前馈控制
**内容**: 在扰动影响输出之前，根据扰动的测量值提前采取控制动作。比反馈控制更主动，但需要预测扰动。

**Applicable_Structure**: 预防性措施、预测性调节、主动干预、扰动补偿

**Mapping_Hint**: 当 Domain A 的反馈延迟太长或扰动可预测时，使用前馈控制——检测扰动源头，提前补偿。

**Case_Study**: 自动驾驶汽车看到前方车辆刹车灯亮起时提前减速，而不是等到距离缩短后才反应。

### 4.  bang-bang 控制 vs 比例控制
**内容**: bang-bang 控制（全开或全关）简单但产生振荡；比例控制（根据误差大小连续调节）更平滑但需要更复杂的设计。

**Applicable_Structure**: 开关决策 vs 渐进调节、离散 vs 连续、简单 vs 精细

**Mapping_Hint**: 当 Domain A 的控制动作过于粗糙时，引入比例控制——根据偏差程度调整响应强度，避免过度反应。

**Case_Study**: 简单的恒温器是 bang-bang 控制（加热开/关），先进的 HVAC 系统使用 PID 控制平滑调节。

### 5. 可控性与可观测性
**内容**: 可控性：能否通过输入将系统从任意状态转移到任意其他状态；可观测性：能否通过输出推断系统内部状态。

**Applicable_Structure**: 干预能力、信息获取、黑箱问题、系统理解

**Mapping_Hint**: 当 Domain A 难以控制时，检查可控性（是否有足够的控制杠杆）；当难以理解时，检查可观测性（是否有足够的信息输出）。

**Case_Study**: 汽车的可控性：方向盘、油门、刹车提供了足够的控制自由度；仪表盘提供了关键状态的可观测性。
