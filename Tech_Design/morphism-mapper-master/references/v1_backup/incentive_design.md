# Domain: Incentive Design
# Source: Charlie Munger, Steven Levitt, Uri Gneezy, John List
# Structural_Primitives: incentive, motivation, crowding_out, principal_agent, alignment

## Core Objects
- **Incentive**: 引导行为的激励因素，包括金钱、声誉、权力、意义等
- **Agent**: 受激励影响的行动者，可能是理性的但信息有限
- **Principal**: 设计激励机制的委托人，希望引导代理人的行为
- **Intrinsic Motivation**: 来自任务本身的满足感，如成就感、好奇心、自主性
- **Extrinsic Motivation**: 来自外部的奖励或惩罚

## Core Morphisms
- **Incentive Alignment**: 设计激励使代理人的自利行为与委托人目标一致
- **Crowding Out**: 外部激励削弱内在动机，导致总动机下降
- **Moral Hazard**: 受保护免受风险后果时采取更冒险行为
- **Adverse Selection**: 信息不对称导致低质量选项驱逐高质量选项
- **Signaling**: 通过激励设计传递类型信息，筛选高能力者

## Theorems / Patterns

### 1. Crowding-Out Effect
**内容**: 当内在动机强烈时引入外部奖励，反而可能降低总体动机和绩效

**Applicable_Structure**: 需要平衡物质激励与意义感、自主性的场景

**Mapping_Hint**: 映射到"创新团队的激励机制"、"社区贡献的奖励设计"、"志愿者组织的管理"

**Case_Study**: 以色列幼儿园对迟到家长罚款后，迟到反而增加，因为家长将迟到视为可购买的服务

### 2. Multitasking Problem
**内容**: 当绩效难以全面衡量时，强激励导致代理人过度关注可衡量指标，忽视不可衡量但重要的方面

**Applicable_Structure**: 多维度绩效但部分难以量化的工作

**Mapping_Hint**: 映射到"教师绩效考核"、"客服人员的服务质量vs效率"、"研发人员的短期vs长期产出"

**Case_Study**: 教师按学生考试成绩获得奖励，导致"应试教育"和忽视品格、创造力培养

### 3. Tournament Theory
**内容**: 相对绩效评估（锦标赛）激励努力，但可能引发破坏对手或过度冒险

**Applicable_Structure**: 团队合作与内部竞争平衡

**Mapping_Hint**: 映射到"销售团队的排名竞赛"、"晋升机制设计"、"合伙人制度"

**Case_Study**: 投行员工的年终奖金高度依赖相对排名，导致过度冒险和内部信息封锁

### 4. Loss Aversion in Incentives
**内容**: 损失厌恶使同等金额的惩罚比奖励产生更强动机，但可能引发焦虑和风险规避

**Applicable_Structure**: 高风险场景下的行为约束

**Mapping_Hint**: 映射到"质量保证的负向激励"、"合规管理的惩戒机制"、"押金制度设计"

**Case_Study**: 工厂安全规定中，对违规的即时小额罚款（损失框架）比安全奖金（收益框架）更有效

### 5. Team Incentives vs Individual Incentives
**内容**: 团队激励促进合作但可能引发搭便车，个人激励激发努力但可能损害协作

**Applicable_Structure**: 需要团队协作的组织设计

**Mapping_Hint**: 映射到"产研团队的考核方式"、"销售与支持的激励平衡"、"跨部门项目奖励"

**Case_Study**: 科技公司发现纯个人提成导致销售隐瞒客户信息，改为团队奖金后信息分享增加但总销售额略降
