# 医疗数字化写作

## 准备

1. 明确文体、受众、用途、长度、截止时间和发布渠道。识别用户是要大纲、草稿、改写还是终稿。
2. 从用户材料中提取核心主张、支持证据、反对意见和希望推动的行动。文章可以有多个相关主张，不强行裁成单一结论。
3. 若材料含患者、医院、商业或未公开项目信息，先确认可使用范围。不要把机密内容发送到外部检索或子代理。
4. 需要匹配用户既有文风时按需读取 `references/my_voice_samples.md`，但不要复制其中与当前任务无关的内容。

## 研究与结构

1. 当前政策、产品、统计数据和行业事件必须联网核验。优先法律法规、政府文件、医院或企业原始公告、标准和论文，并附直接链接与日期。
2. 区分临床结果、运营指标、技术能力和商业推断。没有证据时不要虚构 KPI、患者收益、实施规模或投资回报。
3. 仅在外部研究量大或论证可独立拆分时使用子代理核查来源或执行反方审阅。最终统一检查证据和表述。
4. 组织为清晰的论证链：问题—证据—机制—影响—行动。根据受众保留必要背景，不以攻击性标题替代论证。

## 起草与校验

1. 先给出短大纲；用户已经提供完整结构或要求直接起草时，不增加无必要的确认环节。
2. 使用具体主体、动作、数据和场景，删除空洞口号、重复排比和没有信息量的结尾。
3. 将事实、引用和分析观点明确区分。医学和政策主张保留必要限定，不把相关性写成因果。
4. 交付前检查：事实来源、数字口径、专有名词、患者隐私、政策时效、反方意见和行动可行性。

## 输出与持久化

默认在对话中交付用户要求的成品和必要来源。只有用户要求创建或更新文件时才写入指定输出位置。不要自动保存草稿、发布文章或写入长期记忆；这些动作分别需要明确授权。

保持有判断但不煽动，不使用恐惧、羞辱或心理操纵手法。内容的说服力来自证据、结构和具体后果。
