# 学术科学可视化

## 收集绘图约束

1. 确认数据文件、变量含义、单位、样本量、统计方法、目标期刊和预期图型。
2. 若用户未指定期刊，先按通用出版标准制作，并把待确认的尺寸、字体或格式列为假设。
3. 不猜测缺失数据、统计检验、误差类型或显著性。缺少定义时先询问或清楚标注未知。
4. 需要查询当前期刊要求时，先获得联网授权，并优先引用期刊官方作者指南。

## 设计编码

1. 选择与问题匹配的图型，避免用装饰性图形替代数据关系。
2. 明确颜色、形状、线型和面板标签的语义；不能只靠颜色区分类别。
3. 为每个轴标注变量和单位。截断坐标轴、对数尺度、归一化或平滑必须显式说明。
4. 误差棒必须声明 SD、SEM、CI 或其他定义；显著性标记必须对应已说明的检验和多重比较处理。
5. 多面板图统一字体、刻度、图例和比较尺度，并按阅读顺序安排信息层级。

## 使用本地资源

- 需要期刊尺寸和格式时读取 [references/journal_requirements.md](references/journal_requirements.md) 与 [references/publication_guidelines.md](references/publication_guidelines.md)。
- 需要配色方案时读取 [references/color_palettes.md](references/color_palettes.md)。
- 需要绘图示例时按需读取 [references/matplotlib_examples.md](references/matplotlib_examples.md)。
- 在 Python 环境兼容时，优先复用 [scripts/style_presets.py](scripts/style_presets.py) 和 [scripts/figure_export.py](scripts/figure_export.py)。先查看接口再调用，不假设依赖已经安装。
- 缺少 Python 包、字体、TeX 或外部程序时，报告缺口；只有用户明确授权后才安装或修改环境。

## 渲染与验证

1. 将临时脚本、缓存和试渲染放入当前任务的临时目录；成品写入用户指定的输出目录。
2. 优先同时保留可编辑脚本与矢量格式；光栅格式按期刊要求设置分辨率。
3. 验证：
   - 文件存在且大小大于零；
   - 物理尺寸、DPI 和格式符合目标；
   - 字体、线宽、图例和面板标签在最终尺寸可读；
   - PDF 字体已嵌入或已说明例外；
   - 数值、排序、聚合和统计标记与源数据一致；
   - 灰度和常见色觉缺陷条件下仍可区分。
4. 进行一次视觉检查，排除裁切、遮挡、乱码、过密标签和错误留白。
5. 数据清洗或面板生成可独立拆分且规模较大时，可用子代理并行处理；最终数据口径和统计声明由主代理复核。

## 交付

交付成品图、可复现脚本和简短图注。说明数据变换、误差定义、统计检验、关键设计选择以及仍未满足的期刊要求。不要用确定语气掩盖统计不确定性。
