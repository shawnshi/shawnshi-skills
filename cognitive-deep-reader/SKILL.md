# 深度阅读

## 选择模式

- 使用战略模式处理带明确目标的阅读，例如销售、产品、投资或写作视角。
- 使用结构模式处理逻辑拆解，识别问题、机制和结论变化。
- 目标缺失但不影响主结论时，采用结构模式并说明假设；只有不同目标会显著改变结果时才询问一个关键问题。

## 分析步骤

1. 确定文本试图解决的具体问题，以及作者面对的既有共识或默认做法。
2. 提取核心主张、支持证据、因果链和结论。
3. 做承重墙测试：删除某个概念后，如果论证仍成立，该概念可能只是包装；重新寻找不可替代的机制。
4. 区分事实、作者判断、修辞和解释者推断。
5. 找出最强反例、未证假设、适用边界和证据空白。
6. 战略模式再补充：哪些观点能转化为动作、适用条件是什么、如何用低成本实验验证。

常规文本直接完成。只有超长材料能按章节或来源独立拆分且当前环境支持时，才可用子代理；主代理统一概念、证据和结论。

若用户提供网页链接，可检索该页面以完成阅读；扩展到其他网络来源前先确认。默认不保存文件、不更新记忆。

## 输出

先给三行以内的结论，再按需要展开：

1. 原点：作者面对的具体问题或旧共识；
2. 机制：真正支撑论证的关键变化；
3. 结果：接受该机制后，判断或行动如何改变；
4. 证据：最有分量的数据、案例或推理；
5. 盲区：反例、缺口和不能推出的结论；
6. 行动：仅在战略模式下给出可验证动作。

篇幅随材料和请求调整，不强制固定字数。只有步骤、因果或闭环用图比文字更清楚时才使用一张紧凑图。使用直接、自然的中文，不用内部方法名替代具体含义。
