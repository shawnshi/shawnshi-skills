# Multi-Agent Writer — Audit Checklist (Phase 4)

> Agent 在 Phase 4 审计时必须逐条执行以下检查。任何 FAIL 项必须修复后才能进入 Phase 5。

## A. 结构与逻辑

- [ ] **金字塔检查**: 每章是否以结论开篇，后跟 MECE 的支撑论据？
- [ ] **Action Title 检查**: 每个章节标题是否包含"动词+判断"（如"存量价格战正在摧毁长尾厂商的利润池"），而非名词短语（如"市场竞争现状"）？
- [ ] **MECE 完备性**: 各章之间是否存在语义重叠？是否有明显遗漏维度？
- [ ] **So What 检查**: 每章结尾是否指向明确的"读者下一步行动"或"认知颠覆"？无则强制重写。
- [ ] **论据-推演-定性原子**: 每一段落是否构成完整的"论据 → 推演 → 定性"逻辑原子？

## B. 叙事与文风

- [ ] **心跳节奏检查**: 是否存在连续 3 句以上的等长句？长短句是否交替？
- [ ] **散文体检查**: 是否出现连续超过 3 个 Bullet Points？如有，必须化为散文排比句。
- [ ] **Three-Bold Rule**: 全篇 `**加粗**` 是否不超过 3 处？每处是否为反共识的终极判词？
- [ ] **动词测试**: 是否用具体动作替换了所有"优化、提升、促进、赋能"？
- [ ] **废话前奏测试**: 是否包含"在当今快速发展的时代"、"众所周知"、"综上所述"等零信息密度语句？

## C. 视觉与数据

- [ ] **视觉逻辑检查**: 每个核心章节是否定义了配套的视觉方案（图表/矩阵/流程图）？
- [ ] **数据锚点检查**: 文中是否包含至少 2 个来自真实来源（搜索工具获取）的数据锚点？
- [ ] **图表文本一致性**: 正文是否围绕解释图表来写，而非图文割裂？

## D. 防御与完整性

- [ ] **Red Team 回应**: Phase 1 红队提出的 Top 3 风险，是否在正文中被显性回应或防御？
- [ ] **残余风险披露**: 文末是否包含 `> ⚠️ Residual Risks:` 引用块？
- [ ] **执行摘要**: 文首是否有 150 字以内的 Answer-First 执行摘要？
- [ ] **Anti-Patterns 扫描**: 是否通过了 `ANTI_PATTERNS.md` 的全部检查项？（交叉引用）
