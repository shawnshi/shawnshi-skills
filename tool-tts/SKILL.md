# Text to Speech

## Procedure

1. 确认文本、语言、音色、语速、是否多角色，以及用户要“生成文件”还是“立即播放”。用户已明确要求播放时无需重复确认。
2. 保留用户提供的台词和情绪标签。只有用户要求改写剧本时才调整内容。
3. 检查可用后端：
   - 云端路径需要 `GEMINI_API_KEY`、Google GenAI Python 包和脚本所配置的 TTS 模型。
   - 本地回退需要 Windows、`pyttsx3` 和可用语音。
   - 播放生成音频需要 `ffplay`。
4. 使用 `scripts/tts_engine.py`。将缓存和生成音频写入用户指定目录或当前任务临时目录。
5. 长文本按句子或说话者切分。只有片段独立且并行能力可用时才并行合成；保持原始顺序。
6. 检查生成文件存在、格式可读、片段顺序正确且没有明显截断。播放失败不等于合成失败，应分别报告。
7. 清理临时片段，保留用户要求的最终音频。

## Boundaries

- 不在用户只要求生成文件时自动播放声音。
- 不在未获授权时安装包、设置凭据或上传敏感文本到云端。
- 文本含隐私或机密内容且需要云端合成时，先说明会发送到外部服务并取得同意。
- 不默认保存音色偏好或写入知识库。
- 后端不可用时报告具体依赖，不伪造播放成功。

## Output

交付音频文件路径、所用后端、播放状态和任何降级说明。
