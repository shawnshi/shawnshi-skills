# Video and Long-form Summary

## Procedure

1. 确认输入是 URL、字幕、转录稿还是长文，以及用户需要短摘要、时间轴、观点矩阵还是改写文章。
2. URL 输入优先获取平台提供的字幕、公开转录或用户有权访问的页面内容。无法取得正文时请求用户提供字幕或转录，不根据标题和简介猜测视频内容。
3. 记录视频标题、发布者、发布日期、字幕语言和获取方式。自动字幕可能有错时，标记低置信片段。
4. 按主题或时间段切分材料，提取主张、证据、例子、反例和未回答问题。保留说话者与观点的归属，不把来源观点改写成模型自己的立场。
5. 只有章节可以独立分析且并行能力可用时才并行；合并时去重并统一时间戳。
6. 生成用户要求的格式。需要 2,000 字以上的分析文章时，可按需读取 `resources/synthesis_prompt.md`，但用户的长度、语气和结构要求优先。
7. 逐项回查原文，验证数字、引语、否定词和时间戳。区分来源事实、说话者判断和分析推断。
8. 用户明确要求保存时，将 Markdown 写入指定目录或当前工作区；否则直接交付。

## Boundaries

- 不绕过平台访问控制，不下载用户无权获取的内容。
- 不大段复制受版权保护的字幕或文章。
- 不默认写入知识库或隐藏完整输出。
- 不强制散文、项目符号、攻击性标题或固定观点数量。
- 资料不足时明确说明缺口。

## Output

按用户选择交付：

- 一段式摘要。
- 带时间戳的主题时间轴。
- 主张—证据—反例矩阵。
- 长文分析。
- 来源与不确定性说明。
