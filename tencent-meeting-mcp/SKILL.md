# 腾讯会议操作

## 环境检查

1. 确认 `tmeet` CLI 可用，或确认可以使用 `scripts/mcp_proxy.py` / `scripts/tencent_meeting.py`。
2. 查看当前 CLI 帮助和版本，不依赖文档中写死的参数。需要版本差异时读取 `references/version_management.md`；需要接口字段或错误处理时读取 `references/api_references.md`、`references/error_dictionary.md`。
3. 执行只读状态检查，确认已登录且凭证有效。不得输出、复制或持久化访问令牌。

## 请求处理

1. 提取会议号、主题、时间范围、成员范围和用户希望执行的动作。
2. 时间默认按用户当前时区解释；涉及跨时区参与人时明确列出时区和 ISO 8601 时间，避免猜测相对日期。
3. 查询会议、录制或纪要属于只读操作，可直接执行。创建、更新、取消、邀请成员、改变录制设置等写操作必须先展示：
   - 动作类型
   - 会议主题和会议号
   - 开始与结束时间及时区
   - 参与人或受影响范围
4. 只有用户明确确认上述内容后才执行写操作。确认只对当前列出的动作有效。
5. 查询会议内容时按可用性使用：智能纪要、转写文本、录制元数据。不要为得到文本而自动下载大型录制文件。
6. 分页时遵循当前 CLI 返回的游标字段，记录已见游标并设置最大页数；游标为空、重复或达到限制时停止，避免死循环。
7. 对失败返回实际错误码和可执行的恢复步骤。不要自动重新登录、扩大权限或反复重试写操作。

## 隐私与权限

- 只访问用户有权查看的会议。会议转写、成员信息和录制内容可能含敏感信息，只返回完成请求所需的最少内容。
- 不把会议内容发送到外部服务、长期记忆或文件，除非用户明确要求并确认目标位置。
- 不替用户绕过会议权限、下载限制、成员可见性或组织政策。

## 输出

只读查询返回简洁列表或摘要，并标明时间范围。写操作返回执行结果和可核验的会议标识；失败时不要声称已完成。
