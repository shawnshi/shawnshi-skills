# Healthcare Strategy Workflows

Select only the workflow that matches the requested decision.

## Strategic brief

Use for a bounded management question.

1. Define the decision, audience, time horizon, constraints and success metrics.
2. Build a source table containing facts, dates, assumptions and unknowns.
3. Compare two or three feasible options.
4. Model cost, benefit and risk as ranges.
5. Deliver the recommendation, conditions, residual risks and decisions required.

## Board memo

Use when leaders need a short decision document.

1. Open with the decision and why it is needed now.
2. State verified facts and material unknowns.
3. Present options, financial implications and failure modes.
4. Recommend a path with explicit conditions.
5. End with the decisions, owners and next verification point.

Do not manufacture urgency or force a warning-style opening.

## Deep analysis

Use for multi-source strategy research.

1. Divide the question into policy, workflow, technology, economics and execution.
2. Research each dimension from primary sources.
3. Maintain an evidence table and a separate assumption table.
4. Test the central recommendation against alternatives and adverse scenarios.
5. Assemble the report in sections that can be independently checked.

Use `scripts/blackboard.py` only when a structured state file materially improves a large task. File creation, external publishing and persistence require explicit authorization.

## Validation

Before delivery:

- Verify dates, sources, formulas and units.
- Distinguish clinical outcomes from operational or commercial claims.
- Check that each recommendation has an owner, dependency and stop condition.
- Remove unsupported certainty and manipulative language.
