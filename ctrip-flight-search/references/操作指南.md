# 携程航班查询详细操作指南

## 页面元素定位策略

### 1. 首页导航元素
- **机票按钮**: 使用`backbone`获取页面结构，查找包含"机票"文本的按钮元素
- **元素特征**: 通常具有`aria-label="机票"`或文本内容为"机票"
- **定位方法**: `search(query="机票")` 或通过`backbone`返回的ref直接点击

### 2. 航班查询表单元素
#### 出发地输入框
- **元素特征**: 
  - class包含"depart"或"departure"
  - placeholder文本包含"出发地"或"From"
  - aria-label包含"出发地"
- **定位方法**: 
  ```javascript
  // 使用backbone获取初始结构
  const departureInput = backboneResult.refs.e24; // 示例ref
  
  // 或使用search精确查找
  const departureOptions = search("北京"); // 查找包含城市名的选项
  ```

#### 目的地输入框
- **元素特征**:
  - class包含"arrival"或"destination"  
  - placeholder文本包含"目的地"或"To"
  - aria-label包含"目的地"
- **定位方法**:
  ```javascript
  // 直接搜索目的地城市
  const arrivalOptions = search("上海");
  ```

#### 日期输入框
- **元素特征**:
  - class包含"date"或"calendar"
  - input类型为"text"或"date"
  - 显示格式为YYYY-MM-DD
- **定位方法**:
  ```javascript
  // 搜索特定日期
  const dateElements = search("2026-03-20");
  ```

#### 搜索按钮
- **元素特征**:
  - class包含"search-btn"
  - type="submit"
  - 文本包含"搜索"或"Search"
- **定位方法**:
  ```javascript
  // 搜索按钮文本
  const searchButton = search("搜索");
  ```

## 标准操作流程（SOP）

### 单程查询流程

#### 步骤1: 访问携程首页
```javascript
// 导航到携程官网
navigate("https://www.ctrip.com");
```

#### 步骤2: 进入航班查询页面
```javascript
// 获取页面结构
const homepageStructure = backbone();

// 找到机票按钮并点击
const flightButton = homepageStructure.refs.e16; // 根据实际ref调整
click(flightButton);
```

#### 步骤3: 输入出发地
```javascript
// 获取航班查询页面结构
const searchPageStructure = backbone();

// 在出发地输入框输入城市名
type(searchPageStructure.refs.e24, "北京");

// 等待下拉列表出现后选择具体机场
const beijingOptions = search("北京");
click(beijingOptions[0].ref); // 选择第一个选项（通常是所有机场）
```

#### 步骤4: 输入目的地
```javascript
// 搜索目的地城市选项
const shanghaiOptions = search("上海");
click(shanghaiOptions[0].ref); // 选择第一个上海选项
```

#### 步骤5: 确认日期
```javascript
// 验证日期是否正确设置
const dateCheck = search("2026-03-20");
if (dateCheck.length > 0) {
    // 日期已正确设置
    console.log("日期设置正确");
} else {
    // 需要手动设置日期（此功能待扩展）
    console.log("需要手动设置日期");
}
```

#### 步骤6: 执行查询
```javascript
// 首选方案：使用回车键提交
press("Enter");

// 备选方案：如果回车键无效，点击搜索按钮
// const searchBtn = search("搜索");
// click(searchBtn[0].ref);
```

### 往返查询流程

#### 步骤1-2: 访问携程并进入航班查询页面
（同单程查询步骤1-2）

#### 步骤3: 切换到往返查询模式
```javascript
// 获取查询表单结构
const searchForm = backbone();

// 查找往返选项并点击
const roundTripOption = search("往返");
click(roundTripOption[0].ref);

// 等待表单扩展显示返程日期输入框
wait_for({text: "请选择日期", timeoutMs: 3000});
```

#### 步骤4-6: 输入参数并执行查询
（同单程查询步骤3-6，但表单包含返程日期字段）

### 关键注意事项
- **日期输入限制**: 日期输入框为只读状态，只能通过日历控件选择或使用预设日期
- **表单自动扩展**: 切换到往返模式后，表单会自动显示返程日期输入区域
- **URL结构差异**: 
  - 单程: `/oneway-出发地-目的地`
  - 往返: `/round-出发地-目的地`

## 错误处理和恢复策略

### 常见错误场景

#### 1. 元素不可见或被覆盖
**症状**: 点击操作失败，提示元素被阻挡
**原因**: 日期选择器、城市下拉框等覆盖层未关闭
**解决方案**: 
- 使用键盘操作替代鼠标点击（如Enter键提交）
- 等待覆盖层自动关闭后再操作
- 使用`wait_for`等待特定条件满足

#### 2. 页面结构动态变化
**症状**: 元素引用失效，无法找到之前定位的元素
**原因**: 页面交互后DOM结构重新渲染
**解决方案**:
- 每次交互后重新执行`backbone`或`search`
- 不要依赖之前的元素引用
- 使用稳定的定位策略（如文本内容、class名称）

#### 3. 网络加载延迟
**症状**: 页面加载缓慢，元素未及时出现
**解决方案**:
- 在关键操作前添加适当的等待时间
- 使用`wait_for`等待特定文本或元素出现
- 设置合理的超时时间

### 恢复操作示例

```javascript
// 安全的点击操作封装
function safeClick(elementRef, fallbackAction = null) {
    try {
        click(elementRef);
    } catch (error) {
        if (fallbackAction) {
            fallbackAction();
        } else {
            // 尝试重新获取页面结构
            const newStructure = backbone();
            // 根据新结构重新定位元素
            const newElement = search(elementRef.text);
            if (newElement.length > 0) {
                click(newElement[0].ref);
            }
        }
    }
}

// 安全的表单提交
function safeSubmit() {
    try {
        press("Enter");
    } catch (error) {
        // 回车键失败，尝试点击搜索按钮
        const searchBtn = search("搜索");
        if (searchBtn.length > 0) {
            click(searchBtn[0].ref);
        }
    }
}
```

## 性能优化建议

### 1. 减少不必要的页面操作
- 避免重复的`backbone`调用，只在必要时重新获取页面结构
- 使用`search`进行精确元素查找，而不是遍历整个DOM

### 2. 合理的等待策略
- 在页面跳转后等待必要的加载时间
- 使用条件等待而不是固定延时

### 3. 批量操作优化
- 对于多城市查询，可以复用页面会话
- 缓存常用的城市代码映射关系

### 多城市通用性验证结论
- **城市识别能力**: 携程系统能自动识别中文城市名和对应机场代码
- **操作一致性**: 不同城市组合下的操作流程完全一致
- **稳定性表现**: 键盘回车键提交策略在所有测试城市组合中均有效
- **覆盖范围**: 技能适用于全国主要一二三线城市的任意组合
- **国际支持**: 技能同样适用于国际航线查询，使用相同的表单和操作流程

### 浏览器工具支持的Action清单
- ✅ `navigate`: 访问URL
- ✅ `backbone`: 获取页面结构骨架  
- ✅ `search`: 模糊元素查找
- ✅ `click`: 点击交互
- ✅ `type`: 文本输入
- ✅ `press`: 键盘按键

### 重要限制说明
- ⚠️ **日期输入**: 日期输入框为只读状态，无法直接输入自定义日期
- ⚠️ **日历控件**: 需要通过日历选择器操作，增加了自动化复杂度
- ⚠️ **动态元素**: 页面交互后元素引用频繁失效，需要重新定位
- ⚠️ **国际特殊提示**: 国际航班页面会显示出入境提醒，需在结果解析时考虑
