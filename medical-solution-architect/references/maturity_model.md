# Medical Digital Maturity Model (Strategic Architecture Baseline)

## 0. Level 0: Static (传统手工阶段)
*   **临床**: 纸质文书，口头交付。
*   **管理**: 财务与业务脱节，后置统计。
*   **IT**: 孤立系统，无集成平台，数据烟囱严重。

## 1. Level 1: Digitalized (基础数字化)
*   **临床**: 电子病历（HIS/EMR）上线，以结构化存储为核心。
*   **管理**: 业务系统数据初步沉淀，具备基础运营报表。
*   **IT**: 具备基础集成平台，实现单向数据共享。

## 2. Level 2: Interconnected (互联互通与中台化)
*   **临床**: 实现跨院区、跨系统数据共享。CDSS 辅助诊疗。
*   **管理**: 实时管理看板（BI），初步支持基于数据的精细化管理。
*   **IT**: 遵循 HL7 FHIR 标准，数据中台成型，具备初步的数据治理能力。

## 3. Level 3: Cognitive & Intent-Driven (AI 增强与意图驱动)
*   **架构关键**: 引入 **Medical Semantic Layer (MSL)** 消除语义孤岛。
*   **临床交互**: 从“点击驱动”转向“意图驱动”，具备 **ACI (环境智能)** 感知能力。
*   **意图流映射**: 实现 **T2A (Text-to-Action)** 编排，AI 成为医生的主动协作队友。
*   **管理**: 自动识别医保合规与合规审计风险，实现过程化控费。

## 4. Level 4: Self-Evolving (自进化智慧医院)
*   **核心特征**: 数字化系统具备自愈与自进化能力。
*   **临床**: 精准医学、科研数据与临床诊疗实时闭环，自动化证据链生成。
*   **管理**: 价值医疗（VBC）驱动的分配机制，数字化契约自动执行。
*   **IT**: **Agentic Mesh** 架构，系统根据临床意图动态生成交互界面（GenUI）与业务流。
