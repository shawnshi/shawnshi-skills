# 洞察型标题 (Action Titles) 对比示例

> V5.0 要求：标题必须是动词驱动的业务判词，禁止使用平庸的描述性名词。

## 标题构造公式

```
[动词/动作] + [具体业务对象] + [量化目标或战略判词]
```

**示例推导**：
- "数据治理" → **打破** 科室数据孤岛，**重构** 基于患者时间轴的诊疗流
- "系统升级" → **从** 烟囱式堆砌 **到** 云原生中台化：范式跃迁路径
- "成本控制" → **以** DRG 单病种成本 **为锚点** 的数据穿透

## 标题自检清单

- [ ] 标题长度 ≥ 15 字？（短于此通常意味着描述性标签而非判词）
- [ ] 包含具体动词？（如：打破、重构、压缩、终结、推演）
- [ ] 包含业务领域关键词？（如：DRG、信创、CDSS、电子病历）
- [ ] 读起来像一个结论/判断，而非一个分类标签？
