# Medical Solution Structure Standards (V5.0)

> 对齐 SKILL.md V5.0 的 7 阶段执行协议与核心架构约束。

## Type A: 战略执行摘要 (Executive Action Summary) — 约 2 页
1. **[冲突定义]**：一句话定义医院当前面临的"不可能三角"矛盾。
2. **[战略判词]**：一句话定义方案的架构灵魂与转型方向。
3. **[受众分层价值]**：分别为院长/CIO/临床主任各一句核心收益。
4. **[TCO 与 ROI 概览]**：硬性合规满足路径 + 量化投资回报预估。
5. **[致命风险提示]**：1 个可能导致项目延期的关键风险。

## Type B: 概要方案 (Strategic High-Level Design) — 约 10 页
1. **[现状诊断与痛点议题树]**：MECE 框架下的医院"不可能三角"分析。
2. **[业务蓝图]**（散文体）：数字化如何重塑患者旅程与医护体验。
3. **[应用与数据架构]**（矩阵体）：
    - 核心模块定义表
    - Mermaid 系统交互拓扑图
    - 数据流向图
4. **[信创合规与数据治理]**：
    - 国产化适配清单（OS/数据库/中间件）
    - 等保/加密脱敏标准
    - 主数据管理 (MDM) 规则
5. **[三阶段实施路线图]**：Quick Wins → Strategic Shift → Future Vision。
6. **[旧城改造风险矩阵]**：数据迁移、系统双活并行、第三方厂商接口风险与减缓策略。
7. **[TCO 测算]**：软硬件、接口改造、隐性成本全量估算。

## Type C: 完整技术蓝图 (Detailed Strategic Blueprint) — 30 页以上
1. **[MECE 深度需求解构]**：完整议题树。
2. **[五位一体架构方案]**（全部使用矩阵体）：
    - 业务架构：临床业务流重塑。
    - 应用架构：微服务、API 网关、中台解耦。
    - 数据架构：数据治理、数据中台、知识图谱。
    - 技术架构：云原生、弹性伸缩、高可用。
    - 安全架构：信创适配、等保合规、数据脱敏。
3. **[受众分层价值工程]**：
    - 院长 (50%)：DRG 结余、国考指标、TCO 拐点。
    - CIO (30%)：平滑割接、合规减负、微服务防宕机。
    - 临床主任 (20%)：临床减负、CDSS 拦截差错、文书自动生成。
4. **[精细化业务场景设计]**：
    - 每个场景必须量化：`[当前状态] -> [目标状态] (量化指标)`
    - 门急诊/住院/运营管理/医保控费分场景展开。
5. **[实施治理与旧城改造方案]**：
    - 新老系统双活并行期设计
    - 灰度上线切块策略
    - 科室 Key User 利益绑定
    - 历史数据清洗与迁移方案
    - 第三方厂商接口协调策略
6. **[预算模型与分阶段 ROI 分解]**。
7. **[高管决策备忘录]**（1 页 Answer-First）。

---
### 写作硬约束 (Hard Constraints)
- **Every Heading MUST be an Action Title**：标题必须是动词驱动的业务判词，禁止平庸名词。
- **Every Feature MUST map to a Documented Pain Point**：功能必须追溯至诊断出的痛点。
- **Every Claim MUST be Quantified**：禁止"提高效率"等空泛表述，必须量化为工时/成本/指标。
- **散文体仅限"业务蓝图"章节**：IT 架构、数据流、接口清单强制使用表格/列表/Mermaid。
- **加粗不超过 3 处**：全篇加粗内容必须是直击核心矛盾的战略判词。
- **信创合规与旧城改造必须覆盖**：缺失直接判定为废稿。
