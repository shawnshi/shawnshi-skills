## Mathematical Foundation

**范畴论中的 Initial Object**：
在范畴 C 中，对象 I 称为初始对象，如果对 C 中任意对象 X，存在唯一的态射 I → X。

Initial Object 的特点是：
- 它是结构的"起点"
- 它本身没有内在结构（无法被进一步分解）
- 所有对象都可以从它派生

**禅宗公案逻辑**：
- 某些问题本身存在范畴错误（Category Mistake）
- 如同问"红色有多重"——语法正确，语义荒谬
- 此时提供"答案"是错误回应，应提供"重构"

## Trigger

### 自动触发
- 遍历所有 Domain B 后无有效映射
- 用户问题包含逻辑悖论（如"圆的方"、"万能的石头"）
- 连续 3 次映射验证失败（Commutativity Check 失败）
- 问题结构为空（O_a 或 M_a 无法识别）

### 手动触发
命令：`/morphism-koan`

## Logic

### Step 1: 识别逻辑僵局
检测以下情况：
- **结构不匹配**：Domain A 的结构在 Domain B 中无同构
- **约束冲突**：Domain A 的约束与所有 Domain B 的基本公理矛盾
- **悖论问题**：问题本身违反逻辑一致性

### Step 2: 范畴错误诊断
分析问题是否在错误的范畴中被提问：
- 将"价值问题"当作"技术问题"
- 将"涌现属性"当作"个体属性"
- 将"过程"当作"状态"
- 将"关系"当作"实体"

### Step 3: 生成 Re-framing Question
不提供解决方案，而是重构问题本身：
- 指出隐藏的前提假设
- 建议更换提问范畴
- 提供禅宗公案式的顿悟触发

## Input/Output

### 输出格式

```markdown
### 【公案中断】

**诊断**：
您的问题"[原问题]"存在 [逻辑悖论/范畴错误/结构不匹配]。

**类比**：
这类似于问："[相似悖论]"

**重构提问**：

也许问题不在于 [原问题焦点]，而在于 [新的思考维度]？

> [禅宗公案式顿悟触发，引用 zhuangzi 或 second_order_thinking]

**建议的新问题**：
1. [重构问题 1]
2. [重构问题 2]
3. [重构问题 3]

---

"[引用语]" —— [出处]
```

### 示例输出

```markdown
### 【公案中断】

**诊断**：
您的问题"如何让所有人都喜欢我"存在范畴错误。

**类比**：
这类似于问："如何让水同时向上流和向下流？"

**重构提问**：

也许问题不在于"让所有人喜欢"，而在于"为什么需要被所有人喜欢"？

> 庄子曰："至人无己，神人无功，圣人无名。"当你不再追逐被喜欢，被喜欢反而成为副产品。

**建议的新问题**：
1. 我想被喜欢，是为了填补什么内在缺失？
2. 如果只有 3 个人真正理解我，那足够吗？
3. "被喜欢"是一种状态，还是一个过程？如果是过程，我在哪个阶段？

---

"所有的追求都是逃避，所有的逃避都是追求。" —— 禅宗公案
```

## Integration

**挂载点**：
- Phase 3 (Functorial Mapping) 失败后
- Phase 4.1 (Commutativity Check) 连续失败后
- 作为独立模块手动调用

**作用**：
处理基础流程无法解决的逻辑僵局，提供更高阶的反思框架

**强制执行**：否（可选模块，但推荐在遇到僵局时使用）

**与其他模块的关系**：
- 可在 `yoneda_probe` 之前使用（如果对象本身模糊）
- 可在 `adjoint_balancer` 之前使用（如果问题本身需要重构）
- 与 `limits_colimits` 互补（koan_break 处理"无解"，limits 处理"多解"）
