## Mathematical Foundation

**极限 (Limit)** 与 **余极限 (Colimit)**：
- 极限：从多个对象中提取"最大公共子结构"（交集思维）
- 余极限：从多个对象中提取"最小公共覆盖"（并集思维）

**商业映射**：
当多个领域（Domain B₁, B₂, B₃...）都给出了洞察时：
- 极限 = 这些洞察的**共同核心**（最稳定的结论）
- 余极限 = 这些洞察的**互补整合**（最完整的图景）

## Trigger

### 自动触发
- 用户主动要求"多领域验证"、"交叉验证"
- 连续使用了 3 个或以上 Domain B
- 用户问"这些分析有什么共同点？"
- 发现不同 Domain B 的结论存在冲突需要调和

### 手动触发
命令：`/morphism-limit`（提取共同核心）
命令：`/morphism-colimit`（整合互补洞察）

## Logic

### Step 1: 收集多域洞察
整理各 Domain B 的核心结论：

| Domain B | 核心定理 | 商业洞察 | 建议方案 |
|----------|----------|----------|----------|
| 博弈论 | 信号博弈 | 价格作为质量信号 | 超高端产品线 |
| 进化生物学 | 关键创新 | 技术突破重置格局 | 机械臂技术 |
| 控制论 | 前馈控制 | 预判优于响应 | 技术雷达系统 |

### Step 2: 构造图表 (Diagram)
绘制各洞察之间的关系：
- 哪些是相互支持的（同向箭头）
- 哪些是相互冲突的（反向箭头）
- 哪些是独立的（无连接）

### Step 3: 计算极限 (Limit)
提取**最大公共子结构**：
- 所有 Domain B 都认同的结论
- 跨域最稳定的洞察
- 不受具体领域假设影响的核心逻辑

### Step 4: 计算余极限 (Colimit)
整合**互补洞察**：
- 各 Domain B 的独特贡献
- 构成完整图景的拼图
- 需要组合执行才能生效的配套方案

## Input/Output

### 输入
- 3+ 个 Domain B 的分析结果
- 各领域的推演提案

### 输出格式

```markdown
### 【极限提取】- 跨域元逻辑

#### 1. 多域洞察总览
[表格呈现各 Domain B 的核心结论]

#### 2. 洞察关系图
```
博弈论 ──→ 信号传递 ──┐
                       ├──→ 共同核心 ──→ 元逻辑
进化论 ──→ 创新突破 ──┘
                       │
控制论 ──→ 预判机制 ──┘
```

#### 3. 极限 (Limit) - 最大公共子结构

**跨域共识**（所有领域都支持）：
- 技术领先是核心差异化来源
- 需要建立不可逆的沉没成本
- 长期关系优于单次交易

**稳定性评级**：⭐⭐⭐⭐⭐
（不受具体领域假设影响，普适性最强）

#### 4. 余极限 (Colimit) - 互补整合方案

**Domain B 独特贡献**：
| 领域 | 独特洞察 | 互补性 |
|------|----------|--------|
| 博弈论 | 价格信号机制 | 解决"如何定价" |
| 进化论 | 关键创新时机 | 解决"何时推出" |
| 控制论 | 反馈调节频率 | 解决"如何迭代" |

**整合执行方案**：
将各领域的独特洞察组合成一个完整的行动链条：
1. [来自控制论] 建立技术雷达 → 识别关键创新时机
2. [来自进化论] 在临界点集中投入 → 实现技术突破
3. [来自博弈论] 通过定价信号 → 向市场传递质量信息

#### 5. 元逻辑 (Meta-Logic)

**跨域第一性原理**：
[用一句话概括所有 Domain B 共同指向的核心真理]

**普适性定理**：
如果 Domain A 具有 [特征]，那么在任意 Domain B 中都能找到 [同构结构]

**战略启示**：
基于元逻辑的长期战略方向（不受具体行业变化影响）
```

## Integration

挂载点：多 Domain B 分析完成后
作用：整合跨域洞察，提取最稳定的底层逻辑
