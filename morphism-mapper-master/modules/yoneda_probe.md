## Mathematical Foundation

**米田引理 (Yoneda Lemma)**：
对于任意范畴 C 中的对象 X，有自然同构：
Hom(-, X) ≅ X

**商业映射**：
一个公司的本质 = 它与所有利益相关者的交互方式集合
放弃直接定义，通过"关系拓扑"推断"黑盒"内部结构

## Trigger

### 自动触发
- Domain A 关键对象属性缺失 > 30%
- 用户描述中出现"不清楚"、"不了解"、"不透明"、"神秘"等词
- 对象为公司/产品时，公开信息极少（如未上市公司、新产品）

### 手动触发
命令：`/morphism-yoneda`

## Logic

### Step 1: 收集关系网 (Hom(-, X))
列出所有与目标对象 X 产生交互的实体及其交互方式：

| 关系方向 | 交互对象 | 交互方式 | 信息/价值流向 |
|----------|----------|----------|---------------|
| 输入 | 供应商A | 提供核心零部件 | A → X |
| 输入 | 投资者B | 资金投入 | B → X |
| 输出 | 客户C | 购买产品/服务 | X → C |
| 输出 | 竞争对手D | 对标/模仿 | X → D (间接) |
| 合作 | 合作伙伴E | 联合开发 | X ↔ E |

### Step 2: 构建关系拓扑
- 绘制关系图谱
- 识别关键节点（度中心性最高的实体）
- 识别信息瓶颈（不可替代的交互关系）

### Step 3: Domain B 映射
在 Domain B 中寻找具有相同关系拓扑的对象 Y：
- 相同的输入-处理-输出结构
- 相似的依赖关系模式
- 同构的利益相关者网络

### Step 4: 反向推断
通过 Y 的已知属性，反推 X 的可能属性：
"如果 X 的关系拓扑与 Y 同构，那么 X 很可能具有 Y 的核心特征"
