# Domain: Kaizen
# Source: Taiichi Ohno, Shigeo Shingo, Masaaki Imai, W. Edwards Deming
# Structural_Primitives: continuous_improvement, waste_elimination, standardization, visualization, flow

## Core Objects
- **Value**: 从客户视角定义的真正增值活动
- **Waste (Muda)**: 不增加价值的活动，包括过量生产、等待、运输、加工、库存、动作、缺陷、未被利用的人才
- **Standard Work**: 当前最佳实践的文件化，改进的基准
- **Gemba**: 实际工作现场，价值创造或问题发生的实际地点
- **Kaiaku**: 与改善相反的概念，指恶化或负面改进

## Core Morphisms
- **PDCA Cycle**: Plan(计划) → Do(执行) → Check(检查) → Act(处理) 的持续循环
- **5S**: 整理(Seiri)、整顿(Seiton)、清扫(Seiso)、清洁(Seiketsu)、素养(Shitsuke)
- **Just-In-Time**: 在需要的时间、按需要的数量、生产需要的产品
- **Jidoka (Autonomation)**: 带有人字旁的自动化，机器在异常时自动停机
- **Poka-Yoke**: 防错设计，使错误不可能发生或立即被发现

## Theorems / Patterns

### 1. Continuous Improvement vs Breakthrough
**内容**: 改善强调小步快跑的渐进改进，与激进的大规模变革相对，长期累积效果超过大变革

**Applicable_Structure**: 需要持续优化且难以承受剧变冲击的系统

**Mapping_Hint**: 映射到"产品迭代策略"、"组织文化演进"、"个人技能提升"、"避免颠覆性风险"

**Case_Study**: 丰田生产系统通过每天数百个小改进，数十年累积形成不可复制的竞争优势

### 2. Gemba Walk
**内容**: 管理者必须亲自到现场观察实际工作，而非依赖报告，才能发现真实问题和改进机会

**Applicable_Structure**: 信息在传递中失真的组织决策场景

**Mapping_Hint**: 映射到"管理者一线调研"、"用户访谈vs数据分析"、"实地考察供应链"、"走出办公室"

**Case_Study**: 丰田管理者花费大量时间在车间观察，直接发现问题并与工人共同解决

### 3. Andon Cord (Stop the Line)
**内容**: 任何工人发现质量问题时都有权停止生产线，问题立即解决而非后续返工

**Applicable_Structure**: 质量优先、鼓励暴露问题的组织文化

**Mapping_Hint**: 映射到"质量第一原则"、"鼓励员工报告问题"、"即时修复vs债务累积"、"心理安全感"

**Case_Study**: 丰田工厂中，拉下安灯绳停止生产线不会受罚，反而会被帮助解决问题

### 4. Mura, Muri, Muda
**内容**: 三种浪费：不均衡(Mura)导致过度负荷(Muri)进而产生浪费(Muda)，应同时消除

**Applicable_Structure**: 资源配置和工作负荷管理

**Mapping_Hint**: 映射到"员工工作负荷均衡"、"需求波动管理"、"项目资源平滑"、"避免过度加班"

**Case_Study**: 精益生产通过节拍时间(takt time)平衡生产线，消除瓶颈和闲置

### 5. Nemawashi (Consensus Building)
**内容**: 决策前进行非正式的、一对一的沟通和准备，确保执行时获得广泛支持

**Applicable_Structure**: 需要组织共识和减少变革阻力的决策场景

**Mapping_Hint**: 映射到"跨部门项目协调"、"变革管理"、"重要决策的前期沟通"、"避免表面共识"

**Case_Study**: 日本企业在正式会议前进行大量非正式沟通，正式会议主要是确认已达成共识
