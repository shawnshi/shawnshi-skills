# /morphism-add-domain 命令

## 用途

基于 V2 标准创建新的领域知识文件。

## 触发方式

- `/morphism-add-domain "领域名称"`
- 自然语言："增加易经思想领域"
- 自然语言："morphism-mapper新增中医领域"

## 执行流程

### Step 1: 需求理解

当用户请求新增领域时，收集以下信息：

1. **领域名称**: 用户指定的名称（如"易经思想"、"中医"）
2. **英文标识**: 转换为 snake_case（如 "i_ching"、"tcm"）
3. **核心关注点**: 该领域的主要研究对象和问题
4. **关键学者/著作**: 该领域的代表人物和经典著作
5. **Structural_Primitives**: 5-10个核心概念

### Step 2: 生成领域文件

按照 V2 标准格式创建领域文件：

```markdown
# Domain: [领域名称]
# Source: [学者《著作》, ...]
# Structural_Primitives: [概念1, 概念2, ...]

## Fundamentals (100 基本基石)

### 导语
[100-150字，点破该领域最核心矛盾，冷峻简练宗师口吻]
[示例：博弈论导语] "博弈论研究的不是游戏，而是人类在策略互动中的理性困境。它揭示了一个残酷的真相：个体理性往往导致集体非理性，而最优策略常常存在于对他人预期的精准计算中..."

### 一、哲学观 (18条)
[编号1-18，每条≤20字，有力简练，无常识，宗师口吻]
[示例]:
1. 策略的本质是对他人预期的预期
2. 理性即知道他人也理性
3. 均衡是自我实现的预言
...

### 二、核心原则 (22条)
[编号19-40，每条≤20字，该领域的铁律]
[示例]:
19. 无玩家能通过单方面偏离获益
20. 占优策略独立于他人选择
...

### 三、思维模型 (28条)
[编号41-68，每条≤20字，认知工具箱]
[示例]:
41. 博弈树：序贯决策的可视化表达
42. 策略空间：所有可行行动的集合
...

### 四、关键方法论 (22条)
[编号69-90，每条≤20字，可执行的手段]
[示例]:
69. 逆向归纳：从终点倒推最优路径
70. 前向推导：从起点正向求解均衡
...

### 五、避坑指南 (10条)
[编号91-100，每条≤20字，血泪教训]
[示例]:
91. 警惕将单次博弈结论套用于重复互动
92. 避免假设对手非理性——理性是共同知识
...

---

## Core Objects (14个)

- **[Object 1]**: [一句话定义]
  - *本质*: [一句话本质]
  - *关联*: [关联对象]
...
[共14个Object]

---

## Core Morphisms (14个)

- **[Morphism 1]**: [一句话定义]
  - *涉及*: [涉及对象]
  - *动态*: [动态描述]
...
[共14个Morphism]

---

## Theorems / Patterns (18个)

### 1. [定理名称]
**内容**: [定理详细描述]

**Applicable_Structure**: [适用结构描述]

**Mapping_Hint**: [具体可操作："当Domain A面临[具体情境]时，识别[具体结构]，通过[具体方法]实现[具体目标]"]

**Case_Study**: [案例研究]

---

[共18个Theorem]

## Tags

- [标签1]
- [标签2]
...
```

### Step 3: 质量标准验证

生成后自检：

**Fundamentals检查**:
- [ ] 100条完整，编号1-100
- [ ] 5维度数量正确（18/22/28/22/10）
- [ ] 每条≤20字
- [ ] 导语点破核心矛盾
- [ ] 宗师口吻，冷峻简练

**O/M/T检查**:
- [ ] Objects共14个
- [ ] Morphisms共14个
- [ ] Theorems共18个
- [ ] 每个Theorem有完整的4字段
- [ ] Mapping_Hint具体可操作

**一致性检查**:
- [ ] 术语使用准确
- [ ] 无重复条目
- [ ] 内容有力，无常识

### Step 4: 保存文件

保存到 `references/custom/[domain_name]_v2.md`

### Step 5: 返回确认

向用户确认：
- 领域文件已创建
- 文件路径
- 内容概要
- 可以立即使用

## 质量参考

生成内容质量必须达到以下参考文件同等水平：
- `references/game_theory_v2.md`
- `references/quantum_mechanics_v2.md`
- `references/thermodynamics_v2.md`
- `references/zhuangzi_v2.md`

## 注意事项

1. **不要直接生成常识性内容**（如"易经是古老的中国经典"）
2. **导语必须点破核心矛盾**，不是介绍背景
3. **Mapping_Hint必须具体可操作**，不能泛泛而谈
4. **100条每条必须有力**，引发深层思考
5. **保留 V2 结构完整性**：100基石 + 14O + 14M + 18T
