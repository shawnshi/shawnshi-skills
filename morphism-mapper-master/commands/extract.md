# /morphism-extract 命令

## 用途
快速提取 Domain A 的范畴骨架（Objects, Morphisms, Constraints）。

## 用法
```
/morphism-extract "问题描述"
```

## 示例
```
/morphism-extract "用户留存率持续下降，竞品在抢夺市场"
```

## 输出格式
```markdown
### 【范畴骨架】- Domain A

**问题陈述**: 用户留存率持续下降，竞品在抢夺市场

| 类型 | 元素 | 说明 |
|------|------|------|
| Object | 用户 | 产品的使用者，具有选择自由 |
| Object | 竞品 | 提供替代方案的竞争者 |
| Object | 市场 | 用户和竞品的集合，资源有限 |
| Morphism | 使用 | 用户与产品之间的交互关系 |
| Morphism | 流失 | 用户从本产品转向竞品的过程 |
| Morphism | 抢夺 | 竞品吸引用户的主动行为 |
| Constraint | 注意力有限 | 用户无法同时使用所有产品 |
| Constraint | 转换成本 | 用户切换产品的成本阈值 |

**结构特征**: 竞争性系统，多主体博弈，资源（用户注意力）有限
```

## 执行步骤

1. 解析用户输入，识别核心问题
2. 提取 Objects（实体）
3. 提取 Morphisms（关系/动作）
4. 提取 Constraints（约束条件）
5. 总结结构特征

## 注意事项
- 保持抽象层次一致
- 不要过早引入解决方案
- 关注动态关系而非静态属性
