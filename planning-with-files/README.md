# 文件规划器 (planning-with-files)

像 Manus 一样工作：在应对复杂任务时，利用持久化的 Markdown 文件建立“存放在硬盘里的工作记忆 (Working Memory)”。

## 核心能力
- **文件序列化记忆**：通过在当前项目中创建 `task_plan.md` (计划规划)、`findings.md` (关键发现) 和 `progress.md` (进程记录) 进行严密的流程节点控制。
- **强制行动法则 (2-Action Rule)**：约束每进行 2 次浏览或查询动作便立刻写入发现结果，防止视觉多模态的信息在短暂内存上下文中丢失。
- **三振出局除错机制 (3-Strike Error Protocol)**：系统化排错管理（第一次识别并针对修复，第二次寻找绕过或替代方案，第三次重审根本假设）。
- 附带启动与校验完成 (check-complete.sh) 的自动化挂载脚本环境。

## 使用场景
在任何预计超出 5 步工具调用、具备多层分支研究或需长期规划执行能力的复杂项目中自动激活，或手动输入 `/planning-with-files` 以强制启用工作台模式。
