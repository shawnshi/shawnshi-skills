# 历史档案扫描

## 范围与安全

1. 确认目标目录、文件类型、时间范围、排除规则和期望交付物。只扫描用户明确授权的路径，不沿符号链接或快捷方式扩大范围。
2. 默认只读。不要删除、移动、重命名、覆盖或“清理”原文件；即使内容被判定为低价值，也只在报告中标记。
3. 在读取前识别凭证、密钥、身份信息、医疗记录、财务记录和第三方隐私。除非任务明确需要，不在摘要中复述敏感值；必要时脱敏。
4. 遇到加密、损坏、不支持或权限不足的文件时记录路径和原因，不尝试绕过保护。

## 工作流程

1. 先做浅层盘点：路径、类型、大小、修改时间和可读状态。对超大目录先返回规模估计和抽样结果。
2. 按文件类型选择适合的提取方式，保留源文件路径、页码或行号等出处。不要只保留脱离来源的摘要。
3. 对重复或近似内容进行聚类，区分完全重复、版本演进和主题相似。
4. 提取事实、决策、项目复盘、人物或组织、时间事件和未解决事项。无法确认日期时标注“日期未知”。
5. 仅在文件很多、任务可按目录或格式独立拆分时使用子代理。为每个子任务设定相同输出字段，并避免传递无关敏感材料。
6. 汇总时区分原文事实、编辑性归纳和需要人工确认的推断。

## 输出

- 扫描范围与排除项
- 文件清单和读取失败项
- 主题与时间线
- 重复项和版本关系
- 可保留的知识条目及来源
- 敏感信息或数据质量风险
- 建议的后续处理，但不直接执行删除或迁移

## 持久化

默认在对话中交付报告。用户明确要求生成文件时，写入指定输出位置；用户明确要求导入知识库时，先展示拟导入条目、来源和目标库，再执行。不要把“扫描”解释为自动入库授权。
