# hit-customer-analyst Workflow (V6.0)

## 1. 强制前置预检：长效实体记忆唤醒
在启动全网公开搜索前，必须先完成本地历史上下文缝合：
1. 检索 `C:/Users/shich/.gemini/MEMORY/wiki/Entity_*.md` 或使用 Vector Lake 工具查询该机构/关键人的历史图谱与笔记。
2. 提取并保留该实体过去的偏好底噪、我方与其的历史交互状态。
3. 结合用户提供的 `Target_Intent`，锁定本次侦察的核心方向，摒弃无关背景噪音。

## 2. 外部检索优先级
完成本地图谱记忆唤醒后，按以下顺序补充最新证据：

1. 机构官网
2. 卫健委/政府官网
3. 医院公众号或权威媒体
4. 中国政府采购网/省市采购平台
5. 学会、期刊、论文库、学术活动页面
6. 厂商官网或案例页
7. 其他新闻源

## 2. 四类最小事实覆盖
每份简报至少覆盖以下 4 类事实：

- 机构全景：基建、排名、预算、数字化规划
- 关键人画像：履历、学术门派、原话
- 厂商格局：历史中标、核心系统、既有供应商
- 政治与治理：人大/政协/学会/标准角色

## 3. 厂商核验顺序
涉及 HIS/EMR/集成平台时，按以下顺序核验：

1. 采购公告或中标公告
2. 医院/卫健委正式新闻稿
3. 厂商案例页
4. 行业媒体报道

至少两类独立信源一致，才可写入“现有核心系统与厂商格局”。

## 4. 原话摘录规则
- 优先抓技术判断、管理判断、建设理念
- 直接引用必须保留完整 URL
- 原话不足 20 字不计入有效引文
- 实在找不到时，写 `【信息缺口】未找到高质量公开原话`

## 5. 信息缺口标法
找不到信息时，不允许用猜测填空。统一写法：

```text
【信息缺口】未检索到公开可验证证据；已查渠道：机构官网、采购网、学会页面。
```

## 6. vendor_mode 语义切换
- `winmed`: 可使用“卫宁存量主权”“我方能力映射”
- `neutral`: 必须用中性措辞，如“现有核心系统与厂商格局”“能力映射”
- `custom`: 将“我方”替换为用户指定厂商名

## 8. 交付前自检
交付前至少确认：

- 明确包含了 `Target_Intent` 并据此组织了简报内容
- 所有链接都是完整 URL
- 没有残留 `[来源](URL)`、`[项目]`、`[姓名]` 这类占位符
- 双向认知矩阵输出了有效“认知重合点”和“分歧摩擦点”
- 生成了至少 2 个结合痛点的 `Demo 剧本`
- 绝对禁忌不少于 2 条
