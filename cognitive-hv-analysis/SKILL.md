# 横纵深度研究

## 定义研究框架

1. 明确研究对象、时间范围、地区、比较对象和用户决策目标。
2. 建立统一的比较口径。不能把不同年份、地区、样本或定义的数据直接并列。
3. 列出需要验证的关键问题，而不是先写结论再找材料。

## 获取证据

用户明确要求调研、最新信息或提供外部链接时，可在该范围内检索公开网络；否则在扩大联网范围前确认。优先使用一手资料、官方数据、论文和可追溯记录，再用社区反馈补充使用体验。

记录来源日期、事件发生日期和口径。对数字进行交叉核验；单一来源、转述和公关材料必须标记。

当纵向、竞品和学术证据可以独立搜集且当前环境支持并行时，可选用子代理。为各任务规定相同的时间边界和字段，主代理统一核验。

## 纵向分析

1. 按阶段而非年份流水账组织历史。
2. 每个转折解释：当时约束、可选方案、实际选择、短期结果和长期锁定效应。
3. 区分主动战略、被动应对和事后叙事。
4. 找出持续不变的能力、债务和激励结构。

## 横向分析

1. 只比较同一时期和同一问题上的真实替代方案。
2. 使用一致维度，例如用户、性能、成本、商业模式、分发、治理和生态。
3. 同时解释领先原因、短板和对手没有复制的原因。
4. 用户口碑只能作为体验证据，不能代替市场规模或因果结论。

## 汇合与推演

把历史形成的能力和负担映射到当前竞争位置。未来推演至少给出基准、上行和下行情景，并分别列出触发信号，不把预测写成事实。

## 交付

报告包含：

1. 决策摘要；
2. 纵向阶段与关键转折；
3. 横向比较表；
4. 历史包袱和优势来源；
5. 证据冲突与未知项；
6. 情景推演和监测指标；
7. 可点击的来源清单。

默认在当前对话交付。生成独立文件、写入知识库或持久化研究结论必须由用户明确要求并指定位置。
