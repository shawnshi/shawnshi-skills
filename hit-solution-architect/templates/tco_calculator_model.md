# 医疗 IT 方案 TCO 量化基线与测算模型 (TCO Calculator Baseline)

> **核心原则**：严禁大模型编造缺乏常识的财务数据。所有降本增效的推演必须基于以下基线锚点。

## 1. TCO 测算标准公式
`TCO 收益 = (Old CAPEX + Old OPEX) - (New CAPEX + New OPEX + Migration Cost)`

- **CAPEX (资本性支出)**: 服务器、存储、网络设备、安全设备采购、软件授权费用。
- **OPEX (运营支出)**: 机房能耗、维保费用、系统管理员薪资、年度第三方接口维护费。
- **Migration Cost (迁移与隐性成本)**: 数据清洗外包费、旧系统双轨运行期服务器租赁费、停机引发的门诊量流失（隐性）。

## 2. 基线数据与假设锚点 (Baseline Rules)
在输出方案的 TCO / ROI 章节时，如无客户具体财务报表提供，模型**必须强制声明**以下常识假设，不可随意突破：

- **硬件折旧周期**: 5 年。
- **软件维保费率**: 通常为合同额的 8% - 15% / 年。
- **人力成本单价假设**: 运维工程师成本约 15-20 万/年。
- **临床提效天花板**: 临床医生/护士的工作流提效或文书自动生成节省的时间，**绝对上限设定为 30-50%**。如果声明超过 50%，必须提供真实的 HEOR (卫生经济学) 或顶级三甲医院的具体改造案例证明，否则判定为幻觉。

## 3. 强制表格格式要求
必须使用明确划分阶段的矩阵表呈现成本流，不得用口语化段落概括。

| 成本维度 | 建设期 (Year 0) | 运行期 (Year 1) | 运行期 (Year 2) | 运行期 (Year 3) | 备注与口径假设 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **CAPEX (资本性支出)** | - | - | - | - | 硬件5年折旧 |
| **OPEX (运维与人力支出)** | - | - | - | - | 包含信创数据库高级维保 |
| **隐性/迁移成本** | 双轨运行费 / 数据清洗费 | 第三方接口联调延期补偿 | - | - | |
| **预估降本增益 (DRG/医保节约)** | - | 医保拒付率下降X% | 耗材库存周转率提升 | - | |
| **年度净现金流** | - | - | - | - | |

> 在蓝图方案（blueprint）生成时，必须按照上述格式直接生成测算表，禁止输出“系统可大幅节省隐性支出”等假大空废话。
