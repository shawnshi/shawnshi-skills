# Intelligence Quality Standard (IQS) - 情报质量准则

## 1. Intelligence Levels (情报分级)
*   **L1: Signal (原始信号)**: 未经处理的原始事实或数据（如：一个 GitHub 提交）。
*   **L2: Info (信息)**: 具备上下文的信号（如：一个关于 Agent 架构的新库）。
*   **L3: Insight (洞察)**: 经过推演，识别出对当前战略资产的影响（如：该库将显著降低 MSL 的开发成本）。
*   **L4: Alpha (战略优势)**: 能够直接触发具体动作的非共识判断。

## 2. The "So What?" Audit (价值审计)
每一条录入简报的情报必须通过三段论审计：
1.  **事实 (Fact)**: 发生了什么？
2.  **联结 (Connection)**: 与我现有的项目（`memory.md`）有什么关系？
3.  **推演 (Deduction)**: 结论是什么？我该做什么动作？

## 3. Narrative Discipline (叙事纪律)
*   **禁止使用形容词**：用数据和逻辑取代“重大进展”、“革命性”等虚词。
*   **强制关联历史**：提及新技术时，必须对比其与历史类似技术的演进关系。
*   **反直觉优先**：优先筛选并深入分析那些违背直觉或现有共识的情报。
