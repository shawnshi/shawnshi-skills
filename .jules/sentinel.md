## 2025-05-05 - Command Injection via shell=True
**Vulnerability:** Found multiple instances where `subprocess.Popen` or `subprocess.run` was called with `shell=True`. In one case (`hub_utils.py`), it executed an externally controllable command string (`PIH_LLM_COMMAND` environment variable), creating a severe command injection vulnerability. In other cases, passing a list of arguments with `shell=True` on POSIX systems causes only the first element to execute correctly as a command.
**Learning:** `shell=True` can open up command injection vectors if any part of the command string is user-controllable. Furthermore, using `shell=True` with argument lists on Unix-like environments fundamentally alters how arguments are parsed and can cause subtle bugs.
**Prevention:** Always use `shell=False` for subprocess calls. If executing a command string, parse it into a list first using `shlex.split()`. Never pass unsanitized externally controllable strings into `subprocess` commands.
