## 2024-05-11 - [Command Injection via shell=True in subprocess]
**Vulnerability:** Found multiple instances where `subprocess.run` or `subprocess.Popen` were called with `shell=True`. Some of these used a command string directly, while others passed a list of arguments. When passing a string, it is vulnerable to command injection if external input is involved. When passing a list with `shell=True` on POSIX, only the first element is executed as the command, which can lead to unexpected behavior or injection.
**Learning:** `shell=True` is dangerous and often unnecessary. It should only be used when explicitly required to run shell built-ins or complex shell pipelines, and in such cases, extreme care must be taken to sanitize inputs. Developers may accidentally use it thinking it's the default or necessary for basic commands. Passing lists with `shell=True` on POSIX systems is particularly problematic.
**Prevention:** Always use `shell=False` (which is the default) unless absolutely necessary. When executing commands, pass them as a structured list of arguments rather than a single string. If a string must be parsed into a list, use `shlex.split()`. Never pass unsanitized user input into a command string.
