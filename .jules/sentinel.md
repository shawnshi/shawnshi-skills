## 2025-04-28 - Command Injection via `shell=True` in subprocess calls
**Vulnerability:** Found multiple instances of `subprocess.run`, `subprocess.check_output`, and `subprocess.Popen` using `shell=True` across several scripts (`personal-monthly-insights/core/engine.py`, `personal-monthly-insights/debug_sessions.py`, `scripts/io_engine/gather_context.py`, `personal-intelligence-hub/scripts/hub_utils.py`). This allows command injection if any user-controllable input is ever passed to these commands.
**Learning:** `shell=True` was used for convenience to execute string commands or rely on shell builtins/resolution, which unnecessarily broadens the attack surface and can cause security vulnerabilities in Python.
**Prevention:** Always use `shell=False` and pass command arguments as a structured list of strings. If the command string is dynamic (e.g., from an environment variable), use `shlex.split()` to parse it into a structured list safely before passing it to `subprocess` functions.
