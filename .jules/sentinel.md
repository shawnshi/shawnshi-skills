## 2024-05-23 - Command Injection Risks with shell=True
**Vulnerability:** The codebase uses `subprocess.run` and `subprocess.check_output` with `shell=True` alongside an array of arguments (e.g., `['gemini', '--list-sessions']`). While POSIX systems typically interpret only the first argument as the shell command and the rest as arguments to the shell (meaning it might not execute as intended), using `shell=True` is generally a critical security risk as it can allow command injection if any untrusted input is incorporated into the command string.
**Learning:** Using `shell=True` with an array of arguments is a common anti-pattern that not only poses security risks (command injection) but also behaves differently on POSIX systems (where only the first argument is executed by the shell). Python's `subprocess` module handles command arrays safely by default when `shell=False`.
**Prevention:** Always use `shell=False` (the default) when passing a list of arguments to `subprocess.run`, `subprocess.Popen`, or `subprocess.check_output`. If you must pass a single string command, ensure all dynamic inputs are strictly sanitized or, better yet, use `shlex.split()` to convert the string to a list and keep `shell=False`.
