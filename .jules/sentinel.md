## 2025-02-14 - Fix Command Injection Vulnerabilities in Subprocess Execution
**Vulnerability:** Several Python scripts across multiple packages (`personal-monthly-insights`, `scripts/io_engine`, `personal-intelligence-hub`) executed shell commands using `subprocess.run`, `subprocess.check_output`, and `subprocess.Popen` with `shell=True`. Some commands included variables from user input or environment variables (e.g., `os.environ.get("PIH_LLM_COMMAND")`).
**Learning:** Using `shell=True` can allow malicious actors to execute arbitrary system commands if inputs to the shell command are not fully sanitized, which is a classic Command Injection risk. This is particularly dangerous when external configuration variables dictating command structure are dynamically passed to the shell.
**Prevention:** Avoid `shell=True` whenever possible. Always pass arguments to `subprocess` functions as a structured list of strings instead of a single string. If the command originates as a string, use `shlex.split(command)` to safely tokenize the string into a list before execution with `shell=False`.
