## 2024-05-09 - Command Injection in subprocess calls using list arguments and shell=True
**Vulnerability:** Command injection vulnerability and improper parameter passing on POSIX systems via `subprocess.run` and `subprocess.check_output` calls that pass list arguments (e.g., `['gemini', '--list-sessions']`) together with `shell=True`.
**Learning:** In POSIX environments, passing a list of arguments to `subprocess` with `shell=True` causes only the first list element to be executed as the command. Subsequent elements are passed as arguments to the shell itself, often resulting in silent failures or unintended shell behavior, while exposing the application to command injection if any list element includes unvalidated input.
**Prevention:** Always use `shell=False` when passing arguments as a structured list to `subprocess.run`, `subprocess.Popen`, or `subprocess.check_output`. Only use `shell=True` (if strictly necessary, and avoided when possible) with a single, thoroughly-sanitized string command.
