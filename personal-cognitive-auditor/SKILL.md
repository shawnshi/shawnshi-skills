# 认知与执行复盘

## 工作流程

1. 确认复盘周期、目标和可用材料。只读取用户提供的内容或明确授权的数据源；没有数据时，列出缺口，不补造事实。
2. 按“事实—解释—影响—下一步”分离信息。区分可控行为、外部约束和证据不足项，避免把复杂结果简化为人格判断。
3. 对照用户明确承诺，标记“完成、部分完成、未完成、无法判断”，并引用对应证据。
4. 识别最多三个重复模式，说明证据强度和可能的替代解释。不要把推测写成结论。
5. 提出少量可执行动作；每个动作写明触发条件、完成标准和最小下一步。
6. 仅当材料规模大且可以独立拆分时使用子代理做交叉核对。不要让子代理接触超出任务所需的私人数据。
7. 仅在用户明确要求保存时，调用合适的日记或记忆流程；先展示拟保存内容和目标位置，再执行持久化。

## 输出

按需使用以下结构，不强制填充无数据章节：

- 时间范围与证据来源
- 关键事实
- 承诺对照表
- 重复模式与替代解释
- 风险或数据缺口
- 下一步行动

保持直接、克制和具体。可以指出矛盾，但不得羞辱、辱骂或假装掌握用户动机。健康相关动作只能作为一般生活建议；出现危险症状或持续异常时，建议联系合格医疗人员。

## 资源

- 需要日、周、月或年度模板时，读取 `prompts/` 下对应文件。
- 需要字段含义或交接格式时，按需读取 `references/templates.md`、`references/semantic_layer.md` 或 `references/handoff_contract.md`。
- 生成文件草稿后，可运行 `python scripts/audit_gate.py <draft_file>`；若脚本不可用，人工检查章节、证据引用和未证实判断。

## 边界

- 不自动读取历史对话、日历、健康数据或全局记忆。
- 不把复盘结论写入日记、知识库或其他长期存储，除非用户明确授权。
- 不输出隐藏推理过程；只呈现证据、判断和不确定性。
- 不因追求强硬语气而忽略客观原因、数据质量或安全风险。
