# Medical Digital Maturity Model (Baseline Assessment)

## 0. Level 0: Static (传统手工阶段)
*   **临床**: 纸质文书，口头交付。
*   **管理**: 财务与业务脱节，后置统计。
*   **IT**: 孤立系统，无集成平台。

## 1. Level 1: Digitalized (基础数字化)
*   **临床**: 电子病历（HIS/EMR）上线，结构化程度低。
*   **管理**: 业务系统数据初步沉淀，具备基础报表。
*   **IT**: 具备集成平台基础，实现单向数据流。

## 2. Level 2: Interconnected (互联互通)
*   **临床**: 实现跨院区、跨系统数据共享。CDSS（临床决策支持）介入。
*   **管理**: 实时管理看板（BI），初步支持管理决策。
*   **IT**: 遵循 FHIR/HL7 标准，数据中台成型。

## 3. Level 3: AI-Augmented (AI 增强/主动式干预)
*   **临床**: 生成式 AI 处理文书，主动感知环境（ACI）。
*   **管理**: 预测型分析，自动识别医保合规风险。
*   **IT**: Agentic Mesh 架构，业务 API 化（T2A）。

## 4. Level 4: Self-Evolving (自进化医院)
*   **临床**: 精准医学与临床诊疗实时闭环。
*   **管理**: 价值医疗（VBC）驱动的分配机制。
*   **IT**: 自主感知、自主决策、自主修复的数字底座。
